# 拡張機能の公開  

拡張機能を開発してテストすると、拡張機能を配布する準備が整います。 Microsoft Edge アドオン カタログを使用して拡張機能を配布します。  Microsoft Edge ユーザーの既存の Chromium 拡張機能をリリースするには、既存の Chromium 拡張機能を移植 [する方法に移動します][PortChromiumExtension]。  

拡張機能を Microsoft Edge アドオン カタログに公開して、Microsoft Edge のユーザーが利用できる範囲を拡大します。  この記事では、Microsoft Edge アドオン カタログに拡張機能を提出するプロセスについて説明します。  

## 始める前に  

拡張機能の作業プロトタイプを準備する必要があります。  拡張機能を作成する方法については、「使い始める」の [チュートリアルを参照してください][ExtensionsGettingStarted]。  

拡張機能を Microsoft Edge アドオン カタログに公開するには、パートナー センターでアクティブな開発者アカウントを [使用します][MicrosoftPartnerCenter]。  開発者アカウントをお持ちでない場合は、新しい開発者アカウントを作成します。  新しい開発者アカウントを開き、Microsoft Edge アドオン プログラムに登録するには、[開発者登録] に [移動します][DeveloperRegistration]。  

拡張機能パッケージを表す zip ファイルを作成します。  拡張機能パッケージには、次のファイルを含める必要があります。  

*   拡張機能の名前、簡単な説明、アクセス許可、既定の言語など、詳細を指定する拡張機能マニフェスト。  
*   拡張機能に必要な画像や他のファイル。  

マニフェストの次のフィールドは、ストア登録情報の詳細に自動的に含まれます。  フィールドは、ストア登録情報ページ **では読み取り専用** です。  ストア登録情報ページについては、この記事の後半で説明します。  パッケージをパートナー センターにアップロードする前に、フィールド値がストアの詳細ページの希望する表示と一致する必要があります。  マニフェスト ファイルに必要なコードの例については、マニフェスト ファイルの基本を確認してください。  

*   `Name` フィールド (ストアの詳細ページの **表示** 名) を指定します。  
*   `Description` マニフェスト ファイルのフィールドです。これは、ストアの詳細 **ページ** の簡単な説明です。  拡張機能の一覧の上部に表示する簡単でキャッチーな説明を提供します。  含まれている場合、拡張機能マニフェスト ファイルに指定された簡単な説明がストア登録情報に表示されます。  マニフェスト ファイルに簡単な説明が含まれていない場合は、最初の数行の説明が表示されます。  ストア登録情報ページでコンテンツの繰り返しを回避する簡単な説明を提供します。  

## Microsoft Edge アドオン ストアに拡張機能を提出する  

パートナー センターに拡張機能を [提出するには、次][MicrosoftPartnerCenter]の手順を使用します。  

#### 手順 1: 新しい申請を開始する  

開発者ダッシュボードに [移動し、[][MicrosoftPartnerCenter] 概要] ページで [新しい **拡張機能** の作成] **を選択** します。  

#### 手順 2: 拡張機能パッケージをアップロードする  

[パッケージ **] ページ** を使用して、拡張機能パッケージの zip ファイルをアップロードします。  一度にアップロードできるパッケージは 1 つのみです。  [パッケージ] ページでパッケージのアップロードが成功しなかった場合は、申請を **続行** できない。  

パッケージをアップロードするには、パッケージを選択してアップロード フィールドにドラッグします。 [ファイルの参照] **を選択することもできます**。  パッケージがアップロードされると、パッケージが検証されます。  検証が成功したら、拡張機能の詳細を確認し、[次へ] **を選択して** 続行します。  検証エラーがある場合は、問題を解決し、もう一度アップロードしてみてください。  

#### 手順 3: 可用性の詳細を提供する  

[可用性 **] ページ** で、拡張機能の可用性に関する次の情報を入力します。  

##### 表示  

次のいずれかの表示オプションを選択して、拡張機能が Microsoft Edge アドオン カタログで検出可能な場合に定義します。  

*   `Public` \(default\)  
    パブリックでは、すべてのユーザーが検索、Microsoft Edge アドオン カタログ内の参照、または Microsoft Edge アドオン ストア内の拡張機能の登録情報 URL を使用して拡張機能を検出できます。  登録情報の URL は、パートナー センター ダッシュボードの [拡張機能の概要] ページ **で確認** できます。  
    
*   `Hidden`  
    非表示では、Microsoft Edge アドオン カタログ内の検索結果または閲覧から拡張機能が削除されます。  Microsoft Edge アドオン ストアで非表示の拡張機能を配布するには、リスト URL を拡張機能にユーザーと共有する必要があります。  
    
> [!NOTE]
> 拡張機能の可視性をパブリックから非表示**に変更****できます**。  可視性がパブリックに設定されている間に拡張機能をインストールしたユーザーは、拡張機能へのアクセスを保持し、Microsoft Edge アドオン Web サイトを通じて利用可能にした更新プログラムを受け取る。  

##### 市場  

拡張機能を提供する予定の特定の市場を定義します。  市場の既定の設定は、すべての市場であり、後で追加される将来の市場が含まれます。  特定の市場を選択するには、[市場の変更 **] を選択します**。  個々の市場を切り替え、各**** 市場を除外するか、[すべて選択解除] を選択して、選択した市場を個別に追加します。  

> [!NOTE]
> 拡張機能が提供される市場は変更できます。  ユーザーの市場で利用できる間に拡張機能をインストールしたユーザーは、拡張機能へのアクセスを保持します。  ただし、ユーザーは Microsoft Edge アドオン カタログに提出される今後の更新プログラムにアクセスできます。  

[保存 **] を** 選択して[プロパティ] セクション **に進** む。  

#### 手順 4: 拡張機能のプロパティを選択する  

[プロパティ **] ページで、** 次の情報を入力して拡張機能のプロパティを指定します。  プロパティは、Microsoft Edge アドオン カタログのユーザーに表示されます。  

| 拡張プロパティ名 | 説明 |  
|:--- |:--- |  
| Category \(required\) | 拡張機能を最も説明するカテゴリ。  適切なカテゴリに拡張機能を登録すると、ユーザーは拡張機能を簡単に見つけて、その詳細を理解できます。  |  
| プライバシー ポリシーの要件 \(必須\) | 内線情報が個人情報にアクセス、収集、または送信されるかどうかを示します。  [はい] を選択し、指定**** しない場合、内線は認定手順に合格しない可能性があります `Privacy policy URL` 。  |  
| プライバシー ポリシーの URL | 拡張機能がプライバシーに関する法律や規制に従う方法を伝える有効なプライバシー ポリシー URL。  お客様は、プライバシーに関する法律と規制に従って拡張機能を確保する責任があります。  また、個人情報が内線情報によってアクセス、転送、または収集されている場合は、プライバシー ポリシー URL を提供する責任も負います。  拡張機能にプライバシー ポリシーが必要かどうかを判断するには [、Microsoft Edge][MicrosoftAppDeveloperAgreement] 開発者契約と Microsoft Edge アドオン カタログ開発者ポリシー [に移動します][MicrosoftEdgeAddonsCatalogDeveloperPolicies]。  |  
| Web サイトの URL | 拡張機能に関する追加情報を提供する Web ページ。  Microsoft Edge アドオン カタログ内の拡張機能の Web リストではなく、独自の Web サイトのページ `Website URL` をポイントする必要があります。  この `Website URL` 機能は、拡張機能、機能、その他の関連情報についてユーザーが詳しい情報を得るのに役立ちます。  |  
| サポート連絡先の詳細 | サポート Web ページの URL、またはサポート チームに連絡するメール アドレス。  |  
| 成熟したコンテンツ | 拡張機能に成熟したコンテンツが含まれるか指定するチェックボックス。  拡張機能の評価は、拡張機能の対象ユーザーの適切な年齢グループを決定するのに役立ちます。  拡張機能のコンテンツが成熟したかどうかを判断するには、Microsoft Edge アドオン カタログ開発者 [ポリシーに移動します][MicrosoftEdgeAddonsCatalogDeveloperPolicies]。  |  

[ **保存] を** 選択して、[ストア登録情報 **] セクションに進** む。  

> [!Important]
> 登録時に提出した開発者/組織名、Web サイトの URL、サポート連絡先の詳細は、Microsoft Edge アドオン ストアのユーザーに表示されます。  

#### 手順 5: 拡張機能のストア登録情報の詳細を追加する  

次のセクションで提供される情報は、Microsoft Edge アドオン カタログで登録情報を確認するユーザーに表示されます。  一部のフィールドは省略可能ですが、可能な限り多くの情報を指定する必要があります。  ストアで拡張機能を一覧表示するには、次の詳細が必要です。  

*   **拡張機能** パッケージ内の各言語の説明。  
*   **拡張機能パッケージ内の** 各言語の拡張ストア ロゴ。  
    
> [!NOTE]
> 拡張機能 zip パッケージに記載されている言語の少なくとも 1 つについて、最低限必要なストア登録情報の詳細を入力する必要があります。  Microsoft Edge アドオン カタログのストア登録情報に言語を追加または削除するには、[ストア登録情報****] ページの [言語の追加]**ドロップダウンを使用**します。  さらに、言語の詳細ページの [重複する機能] ボタンを使って、1 つの言語から他の言語にアセットを複製することもできます。  

| 言語の詳細プロパティ名 | 説明 |  
|:--- |:--- |  
| 表示名 \(必須\) | 拡張機能 `name` のマニフェスト ファイルで指定された拡張機能の。  提出後にストアの表示名を変更するには、マニフェスト ファイルの名前を更新し、新しい拡張機能パッケージを作成してから、再アップロードします。  |  
| Description \(required\) | このフィールドでは、拡張機能の機能、ユーザーが拡張機能をインストールする理由、またはユーザーが知る必要があるその他の関連情報について `description` 説明します。  10,000 文字未満である必要があります。  |  
| Extension Store ロゴ \(必須\) | 会社を表す画像、または縦横比が 1 で、推奨サイズが `extension logo` 300 x 300 ピクセルの画像。  また、重複するボタンを使って、1 つの言語から他のすべての言語にアセットをコピーすることもできます。  このボタンは、言語のロゴをアップロードした後、フィールドの後に表示されます。  |  
| 小さいプロモーション 用タイル \(オプション\) | この `Small promotional tile` 画像は、ストア内の他の拡張機能と共に拡張機能を表示するために使用されます。  画像のサイズは 440 x 280 ピクセルである必要があります。  また、重複するボタンを使って、1 つの言語から他のすべての言語にアセットをコピーすることもできます。  このボタンは、言語のプロモーション タイルをアップロードした後、フィールドの後に表示されます。  |  
| Screenshots \(optional\) | 拡張機能の機能について詳しく説明する最大 10 `screenshots` 件を提出できます。  スクリーンショットのサイズは、640 x 480 ピクセルまたは 1280 x 800 ピクセルである必要があります。  また、重複するボタンを使って、1 つの言語から他のすべての言語にアセットをコピーすることもできます。  このボタンは、言語の少なくとも 1 つをアップロードした後、フィールドの後に表示されます。|  
| 大きなプロモーション 用タイル \(オプション\) | `Large promotion tiles` は、Microsoft Edge アドオン Web サイトで拡張機能を目立つように機能するためにストアで使用されます。  送信された画像は、ユーザーに表示されます。  PNG ファイルのサイズは 1400 x 560 ピクセルである必要があります。  また、重複するボタンを使って、1 つの言語から他のすべての言語にアセットをコピーすることもできます。  このボタンは、言語のプロモーション タイルをアップロードした後、フィールドの後に表示されます。  |  
| YouTube ビデオ URL \(optional\) | 拡張機能のプロモーション用の YouTube ビデオを含めることはできません。  ビデオ `YouTube video URL` は、拡張機能のストア登録情報ページに表示されます。  |  
| 簡単な説明 \(必須\) | 編集するには、拡張機能パッケージのマニフェスト ファイルの説明フィールドを更新し、再 `short description` アップロードする必要があります。  |  
| 検索用語 \(optional\) | `Search terms` は、ユーザーが Microsoft Edge アドオン カタログで検索するときに拡張機能を検出するのに役立つ単一の単語または語句です。  検索用語はユーザーに表示されません。  |  

##### YouTube ビデオ URL の要件  

ビデオが次の要件を満たしていることを確認します。  

*   YouTube ビデオの内容が Microsoft Edge アドオン カタログ開発者ポリシー [に従っている必要があります][MicrosoftEdgeAddonsCatalogDeveloperPolicies]。  
*   ビデオの広告をオフにします。  詳しくは、「埋め込みビデオ [で既定の広告][GoogleYoutubeAnswer2531367Topic7072227] 形式と広告を設定 [する」をご覧ください][GoogleYoutubeAnswer132596]。  
*   ビデオの埋め込み機能を有効にする。  詳細については、再生リストを [埋め込む&移動します][GoogleYoutubeAnswer171780]。  
    
ビデオの YouTube ビデオ URL を送信するには、次の手順を実行します。  

1.  YouTube で、ストア登録情報ページに追加するビデオを探します。  
1.  ビデオの下で、[共有埋め込**み] を**  >  **選択します**。  
1.  表示される HTML コードをコピーします。  
1.  ストア登録情報の詳細ページで、フィールドに HTML コードを貼り付 `YouTube video URL` けます。  
    
##### 検索用語の要件  

検索用語は、次の要件を満たしている必要があります。  

*   最大 21 語まで使用する検索用語を入力できます。  1 つの単語、語句、または両方の組み合わせとして使用される場合でも、最大 21 単語までしか使用できません。  
*   1 つの単語または語句の最大 7 つの検索用語。  各検索用語の文字制限は 30 文字です。  
    
#### 手順 6: 提出を完了する  

[拡張機能 **の提出] ページで** 、拡張機能のテストに役立つ認定用のメモを追加します。  

##### 認定に関するメモ (省略可能)  

拡張機能を提出する場合は、[認定**** メモ] ページを使用して、認定テスト担当者に追加情報を提供します。  追加情報は、拡張機能が正しくテストされていることを確認するのに役立ちます。  拡張機能が完全にテストされていない場合は、認定に合格しない可能性があります。  

必要に応じて、次の情報を含める必要があります。  

*   テスト アカウントのユーザー名とパスワード。  
*   非表示またはロックされた機能にアクセスする手順。  
*   地域または他のユーザー設定に基づく機能の予想される違い。  
*   申請が既存の拡張機能の更新プログラムである場合は、拡張機能に加えた変更に関する情報を含めます。  
*   申請について審査担当者が理解する必要がある追加情報。  

情報を提供した後、[発行] **を選択** して拡張機能を Microsoft Edge アドオン カタログに提出します。  提出物は認定手順に進みます。  認定プロセスは、申請後最大 7 営業日かかる場合があります。  

申請が認定に合格すると、拡張機能は Microsoft Edge アドオン カタログに公開されます。  パートナー センター ダッシュボードの拡張機能の状態が次に変わります `In the Store` 。  

> [!NOTE]
> 提出または登録プロセスで問題が発生した場合は、[拡張機能][ExtensionsSupportForm]の新しいサポート要求にサポート チケットを提出するか、メールを送信して[ext_dev_support@microsoft.com。][MailtoExtDevSupportMicrosoftCom]
