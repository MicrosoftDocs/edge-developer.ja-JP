# Microsoft Edge アドオン カタログの開発者ポリシー  

## このドキュメントの概要と目的  

Microsoft Edge アドオン カタログの拡張機能の開発に関心をお持ちのお客様に感謝します。  Microsoft Edge アドオン カタログ開発者ポリシー \(Addons Catalog Developer Policies\) は、パートナー センターを通じた拡張機能の[][MicrosoftPartnerCenter]提出や、Microsoft Edge アドオンを通じた拡張機能のプロビジョニングなど、拡張機能に適用されます。  

## 原則  

開始するには、いくつかの原則を使用します。  

*   Microsoft Edge の拡張機能内で、独自の価値を提供する必要があります。  Microsoft Edge アドオン カタログ \(Microsoft Edge アドオン\) から拡張機能をダウンロードする魅力的な理由を提供します。  
*   拡張機能の機能、提供者などについて、共同ユーザーに誤解を与えることはできません。  
*   ユーザー、システム、またはエコシステムをだますのを試みる必要があります。  Microsoft Edge アドオンには、どのような種類の詐欺にも対応する場所はありません。評価とレビュー操作、クレジット カード詐欺、その他の不正なアクティビティなどです。  
    
Microsoft Edge アドオン ストアの開発者ポリシーに従って、拡張機能の魅力と対象ユーザーを強化する選択を行うのに役立ちます。  

拡張機能は、何億人ものユーザーのエクスペリエンスにとって非常に重要です。  お客様が作成した機能を体験して、拡張機能を世界に提供するために楽しみに思っています。  

## 1. 製品ポリシー  

### 1.1 Distinct Function & Value;正確な表現  

拡張機能と関連するメタデータは、記述するソース、機能、および機能を正確かつ明確に反映する必要があります。  

#### 1.1.1 拡張機能には 1 つの目的が必要  

拡張機能には、狭い機能を持つ単一の目的が必要です。  

#### 1.1.2 拡張機能について説明する  

拡張機能のすべての側面は、拡張機能の機能、機能、および重要な制限 (必須またはサポートされている入力デバイスを含む) を正確に記述する必要があります。  最初の実行エクスペリエンスでは、拡張機能の価値提案が明確である必要があります。  拡張機能は、他の拡張機能と同様の名前やアイコンを使用することはできません。また、その表現を行う権限を持たしていない場合は、会社、政府機関、その他のエンティティを表す必要があります。  

#### 1.1.3 機能  

拡張機能は完全に機能している必要があります。  

#### 1.1.4 検索と検出  

検索用語は 7 つの一意の用語を超えることはできません。また、拡張機能に関連する必要があります。  

#### 1.1.5 適切な詳細を提供する  

拡張機能に関する明確で情報に基づいた詳細と、拡張機能の \(metadata\) を一覧に表示する機能を提供する必要があります。  拡張機能は、貴重で質の高いユーザー エクスペリエンスを提供する必要があります。  拡張機能は、Microsoft Edge アドオンでもアクティブなプレゼンスを持っている必要があります。  

#### 1.1.6 安定性とパフォーマンス  

拡張機能が Microsoft Edge のパフォーマンスや安定性に悪影響を及ぼす可能性はありません。  

#### 1.1.7 難読化  

コードを難読化した拡張機能は許可されません。  これには、拡張機能パッケージ内のコードと、Web からフェッチされた外部コードまたはリソースが含まれます。  確認できない場合は、コードの一部をリファクタリングする必要があります。  

#### 1.1.8 ブラウザー設定の変更  

拡張機能では、アドレス バー検索プロバイダーと提案、スタートページまたはホーム ページ、新しいタブ ページ、お気に入りの追加または削除を含む(これらに限定されない)、適切なユーザーの同意なしに、ブラウザーの機能や設定を変更または変更または表示しることはできません。  

ブラウザーの設定を変更した場合は、拡張機能の説明に明示的に文書化する必要があります。  

拡張機能では、Microsoft の Web ページまたはサービスをサード パーティの検索エンジンの設定 (サード パーティの検索エンジンの使用が必要な場合や、ホーム ページをサードパーティの Web プロパティに設定する場合など) に置き換えるキー設定を変更する場合のみ、そのようなサード パーティに採用されている場合や関連付けられている場合に限り、キー設定を変更できます。  

### 1.2 セキュリティ  

拡張機能は、ユーザーのセキュリティや、デバイス、システム、または関連システムのセキュリティや機能を危険にさらしたり損なうのを防げることはできません。  

#### 1.2.1 コンテンツ セキュリティ ポリシー  

> [!NOTE]
> 上記の機能を超えて拡張機能に変更を加えた場合、コードに対する変更は Microsoft Edge コンテンツ セキュリティ ポリシーに準拠 [している必要があります][MicrosoftEdgeContentSecurityPolicyRemoteScript]。  たとえば、拡張機能はリモート スクリプトをダウンロードし、その後、説明されている機能と一致しない方法でスクリプトを実行する必要があります。  

#### 1.2.2 望ましくない悪意のあるソフトウェア  

お客様の拡張機能には、Microsoft の望ましくないソフトウェアと悪意のあるソフトウェアに関する条件で定義されているマルウェアを含めることはできません。また、有効 [にしることはできません][MicrosoftIdentifiesMalwareUnwantedApplications]。  

#### 1.2.3 他のソフトウェアとの依存関係  

説明に依存関係を開示する場合、拡張機能は統合されていないソフトウェア \(別の製品、モジュール、サービス\など) に依存して主要な機能を提供する場合があります。  

#### 1.2.4 Extensions Update  

Microsoft が特に許可しない限り、拡張機能は Microsoft Edge アドオンを通じてのみ更新する必要があります。  

### 1.3 製品はテスト可能  

拡張機能はテスト可能である必要があります。  以下の項目を含め、何らかの理由で拡張機能をテストできない場合は、拡張機能でこの要件が満たされない可能性があります。  

#### 1.3.1 ユーザー資格情報  

拡張機能でログイン資格情報が必要な場合は、フィールドを使用して有効なデモ アカウントを指定 `Notes for certification` します。  

#### 1.3.2 サービスの可用性  

拡張機能でサーバーへのアクセスが必要な場合は、サーバーが正常に動作するようにサーバーが機能している必要があります。  

### 1.4 使いやすさ  

拡張機能は、以下のサブセクションに記載されている機能を含む(これらに限定されない)、使いやすさに関する Microsoft Edge アドオン ストアの標準を満たす必要があります。  

#### 1.4.1 プラットフォーム間の互換性  

拡張機能は、ダウンロードされる可能性のあるすべてのデバイスとプラットフォームで Microsoft Edge と互換性がある必要があります。  互換性がないデバイスに拡張機能がダウンロードされた場合、拡張機能と互換性を持つには、起動時にデバイスが満たす必要がある要件を詳しく説明するメッセージをユーザーに表示する必要があります。  

#### 1.4.2 ユーザー エクスペリエンス  

拡張機能は、速やかに起動し、ユーザー入力に対する応答性を高める必要があります。  拡張機能は引き続き実行され、ユーザー入力に応答し続ける必要があります。  拡張機能は正常にシャットダウンし、予期せず閉じることはできません。  拡張機能は例外を処理し、例外が処理された後もユーザー入力に対する応答性を維持する必要があります。  

### 1.5 個人情報  

以下の要件は、個人情報にアクセスする拡張機能に適用されます。  個人情報には、個人を識別または使用できる、またはそのような情報やデータに関連付けられているすべての情報またはデータが含まれます。  

#### 1.5.1 必要な場合にのみ個人情報を収集する  

お客様の内線情報は、個人情報 \(Web 閲覧アクティビティを含む\) を収集、アクセス、使用、または送信する場合があります。重要な公開されたユーザー向け機能で使用するためにのみ必要な場合にのみ使用します。  

#### 1.5.2 プライバシー ポリシーを維持する  

内線情報が個人情報にアクセス、収集、または送信されるかどうかに関係なく、法律で要求されている場合は、プライバシー ポリシーに関する目立つ通知を提供し、準拠する必要があります。  プライバシー ポリシーは、内線情報によってアクセス、収集、または送信される個人情報、その情報の使用方法、保存方法、およびセキュリティ保護方法をユーザーに通知し、それが開示される関係者の種類を示す必要があります。  プライバシー ポリシーでは、ユーザーが情報の使用と共有に関して持っているコントロール、ユーザーが情報にアクセスする方法、および適用される法律や規制に準拠する必要があります。  拡張機能に新しい機能を追加する場合は、プライバシー ポリシーを最新の状態に保つ必要があります。  

お客様が Microsoft にプライバシー ポリシーを提供する場合、お客様は、Microsoft がこのようなプライバシー ポリシーを拡張機能のユーザーと共有することに同意します。  

#### 1.5.3 サード パーティとのデータの共有  

拡張機能のユーザーの個人情報は、そのユーザーからオプトインの同意を得た後にのみ、拡張機能または関連するメタデータを通じて外部サービスまたはサードパーティに公開することができます。  オプトイン同意とは、要求されたアクティビティに対する拡張機能のユーザー インターフェイスで、ユーザーが次の操作を行った後に、そのユーザーに明示的なアクセス許可を付与することを意味します。  

*   情報へのアクセス、使用、または共有の方法をユーザーに説明し、公開先の関係者の種類を示し、  
*   後でアクセス許可を取り消してオプトアウトするオプションを持つ拡張機能ユーザー インターフェイスのメカニズムをユーザーに提供します。  
    
#### 1.5.4 ユーザー以外の情報の共有  

内線またはメタデータを通じて個人の個人情報を外部のサービスまたはサードパーティに公開するが、情報が共有されているユーザーが拡張機能のユーザーではない場合。  

1.  お客様は、その個人情報を公開するために書面による明示的な同意を得る必要があります。  
1.  情報が共有されているユーザーが、いつでもその同意を取り下げるのを許可する必要があります。  
1.  プライバシー ポリシーでは、この方法で個人情報を収集できると明確に開示する必要があります。  
1.  適用される法律で要求される場合、お客様は、この方法で収集した情報を持つ個人を含む、要求に応じて個人の個人情報を削除する必要があります。  
1.  拡張機能がユーザーに別のユーザーの個人情報へのアクセスを提供する場合は、この要件も適用されます。  
    
#### 1.5.5 情報を安全に送信する  

内線情報が個人情報を収集、保存、または送信する場合これは、最新の暗号化方法を使用して安全に行う必要があります。  

#### 1.5.6 機密性の高い情報  

拡張機能の機能に関連する情報がない限り、拡張機能は、健康データや財務データなどの機密性の高い個人情報を収集、保存、または送信しることはできません。  また、拡張機能は、そのような情報を収集、保存、または送信する前に、ユーザーの明示的な同意を得る必要があります。  

### 1.6 アクセス許可  

拡張機能は、機能するために必要なアクセス許可のみを要求する必要があります。  拡張機能のしくみの説明を提供する必要があります。  拡張機能は、説明に従ってのみ実行する必要があります。  拡張機能は、宣言された通りに実行して機能するために必要な機能を超える機能のアクセス許可を要求しない場合があります。  

### 1.7 ローカライズ  

拡張機能がサポートする必要があるすべての言語について、拡張機能をローカライズする必要があります。  拡張機能の説明のテキストは、宣言する各言語にローカライズする必要があります。  
一部の機能がローカライズバージョンで利用できない程度に拡張機能がローカライズされている場合は、拡張機能の説明にローカライズの制限を明確に示すか、表示する必要があります。 拡張機能によって提供されるエクスペリエンスは、サポートしているすべての言語でかなり似ている必要があります。  

### 1.8 財務トランザクション  

製品に製品内購入、サブスクリプション、仮想通貨、請求機能が含まれる場合、または財務情報を取得する場合以下のセクションの要件が適用されます。  

#### 1.8.1 有料機能  

拡張機能を使用すると、サード パーティの購入メカニズムまたは API を通じて購入したデジタル コンテンツやサービスをユーザーが利用できる場合があります。  

物理的な商品やサービスを購入するには、セキュリティで保護されたサード パーティの購入 API を使用する必要があります。  現実世界での侵害や支援を含む他のサービスに関連して行われた支払いには、セキュリティで保護された第三者購入 API を使用する必要があります。  

*   拡張機能を使用して、支援を受け、または収集したり、プロモーションの一部を実施したりする場合は、適用される法律に従って実施する必要があります。  
*   また、Microsoft がプロモーションのスポンサーでなければならないと明確に示す必要があります。  
*   拡張機能で販売される製品内製品は、法的に有効な通貨 (USD、ユーロなど) や物理的な商品やサービスに変換しることはできません。  
    
セキュリティで保護されたサード パーティの購入 API の使用には、次の要件が適用されます。  

*   取引の時点、またはユーザーから支払い情報または財務情報を収集した時点。拡張機能は、商取引プロバイダーを識別し、ユーザーを認証し、トランザクションのユーザー確認を取得する必要があります。  商取引プロバイダーは、金融交換用の安全なプラットフォームを維持しています。  
*   拡張機能によって、この認証を保存できる機能がユーザーに提供される場合がありますが、ユーザーは、すべてのトランザクションで認証を要求するか、製品内トランザクションをオフにする必要があります。  
*   拡張機能がクレジット カード情報を収集する場合、またはクレジット カード情報を収集するサード パーティの支払処理者を使用する場合、支払い処理は現在の PCI Data Security Standard \(PCI DSS\) を満たす必要があります。  
    
#### 1.8.2 有料機能の公開  

拡張機能と関連するメタデータは、提供される製品内購入の種類と価格の範囲に関する情報を提供する必要があります。  ユーザーに誤解を与え、試用エクスペリエンスの範囲と条件を含め、製品内のプロモーションや提供物の性質を明確にしなけらねない。  試用中または試用後にユーザーが作成したコンテンツへのアクセスを拡張機能が制限している場合は、事前にユーザーに通知する必要があります。  さらに、拡張機能では、拡張機能で購入オプションを開始しているユーザーに明確にする必要があります。  

### 1.9 通知  

拡張機能では、通知のシステム設定を尊重する必要があります。  つまり、ユーザーに対する広告や通知の表示は、通知が Microsoft プッシュ通知サービス \(MPNS\)、Windows プッシュ通知サービス \(WNS\)、その他のサービスによって提供されるかどうかに関係なく、ユーザー設定と一致している必要があります。  ユーザーが製品固有またはシステム全体の通知を無効にした場合、拡張機能は機能し続けなければならない。  

製品が MPNS または WNS を使用して通知を送信する場合は、次の要件に準拠する必要があります。  

#### 1.9.1 一般的なガイダンス  

WNS または MPNS を通じて提供される通知は製品コンテンツと見なされ、すべてのアドオン カタログ開発者ポリシーの対象です。  

#### 1.9.2 通知の所有権  

拡張機能から開始された通知のソースをあいまいにしたり偽装したりしることはできません。  

#### 1.9.3 機密情報または機密情報なし  

通知に含めずに、ユーザーが機密または機密と合理的に考慮する可能性がある情報は含めずにしてください。  

#### 1.9.4 通知の目的  

拡張機能から送信される通知は、その拡張機能または Microsoft Edge アドオン カタログで公開する他の拡張機能に関連する必要があります。また、拡張機能に関連しないプロモーション メッセージを含めることはできません。  

### 1.10 広告の行為とコンテンツ  

すべての広告関連アクティビティには、次の要件が適用されます。  

#### 1.10.1 目的  

拡張機能の主要なコンテンツは広告にし、広告は拡張機能の他のコンテンツと明確に区別できる必要があります。  

#### 1.10.2 ポリシーと契約  

拡張機能に表示される広告コンテンツは、Microsoft クリエイティブ受け入れポリシー [に従う必要があります][MicrosoftAdvertisingCreativeAcceptancePolicies]。  
拡張機能に広告が表示される場合、表示されるコンテンツはすべて、アプリ開発者契約とこのポリシーの広告 [要件に準拠][MicrosoftAppDeveloperAgreement] している必要があります。  

#### 1.10.3 広告の品質  

*   拡張機能の主な目的は、ユーザーに広告をクリックしてみることです。  
*   拡張機能は、表示される広告の可視性、価値、品質を妨げるものや、低下したりしないようにする必要があります。  

#### 1.10.4 プロモーション  

パートナー センターの広告キャンペーン機能を通じて拡張機能を宣伝するプロモーション用の広告キャンペーンを購入[][MicrosoftPartnerCenter]または作成する場合、関連するランディング ページを含め、Microsoft に提供する広告資料はすべて[、Microsoft クリエイティブ][MicrosoftAdvertisingCreativeSpecifications]仕様ポリシーと[Microsoft クリエイティブ][MicrosoftAdvertisingCreativeAcceptancePolicies]受け入れポリシーに準拠している必要があります。  

#### 1.10.5 ユーザーに広告のOpt-Out通知Interest-Basedする  

プライバシーに関する声明または使用条件は、広告サービス プロバイダーに個人情報を送信する予定をユーザーに通知し、関心に基づく広告をオプトアウトする方法をユーザーに伝える必要があります。  

#### 1.10.6 その他のガイドライン  

内線が 13 歳未満の子供を対象にしている場合は、子[のオンライン][FTCChildrensPrivacy]プライバシー保護法で定義されています。パートナー センターでこの事実を[][MicrosoftPartnerCenter]Microsoft に通知し、拡張機能に表示される広告コンテンツが 13 歳未満の子供に適している必要があります。  

## 2 コンテンツ ポリシー  

次のポリシーは、Microsoft Edge アドオンで配布するために提供されるコンテンツとメタデータ \(発行元名、内線番号の名前、拡張機能アイコン、拡張機能の説明、拡張機能のスクリーンショット、拡張トレーラーとトレーラーのサムネイル、その他の拡張メタデータを含む)に適用されます。  コンテンツとは、拡張機能に含まれる画像、サウンド、ビデオ、テキスト、拡張機能を通じて公開されるタイル、通知、エラー メッセージ、広告、およびサーバーまたは拡張機能が接続するサーバーから配信されるすべてのコンテンツを意味します。  拡張機能と Microsoft Edge アドオンは世界中で使用され、地域的および文化的な規範のコンテキストでこれらの要件が解釈され、適用されます。  

### 2.1 Microsoft Edge アドオン カタログ一覧のコンテンツ要件  

拡張機能に付随して提出するメタデータなどのコンテンツには、成熟したコンテンツが含まれている可能性があります。  
Microsoft Edge アドオン ストア登録情報の要件を満たしていない申請は拒否されたか、速やかに削除されます。  

### 2.2 名前、ロゴ、オリジナル、およびサード パーティを含むコンテンツ  

拡張機能および関連するメタデータ内のすべてのコンテンツは、お客様が最初に作成するか、第三者の権利者から適切にライセンスを取得する必要があります。また、権利者が許可した場合、または法律で許可されている場合にのみ使用する必要があります。  

### 2.3 危害のリスク  

#### 2.3.1 要件  

拡張機能には、現実世界での活動を促進または美化するコンテンツ (\(a\) 極端または不当な暴力) を含めることはできません。\(b\) 違反\(c\) 不正な武器の作成または \(d\) 人、動物、または実性または個人のプロパティに対する武器の使用。  

#### 2.3.2 責任  

お客様の内線番号は、\(a\) エンド ユーザーまたは他の人または動物に対して安全上のリスクを与え、また、不快感、けが、その他の損害を与えることはできません。または \(b\) は、実際または個人のプロパティに損害を与えるリスクまたは損害を与える可能性があります。  お客様は、すべての拡張機能の安全性テスト、証明書の取得、および適切な機能保護の実装に関する責任を負います。  プラットフォームの安全性や快適さを備える機能を無効にしたり、法的に必要なすべての警告や業界標準の警告、通知、免責事項を拡張機能に含める必要があります。  

### 2.4 中傷、中傷、中傷、中傷、および脅威  

内線に中傷、または危うい内容を含めることはできません。  

### 2.5 不快コンテンツ  

拡張機能と関連するメタデータには、機密性の高いコンテンツや不快なコンテンツを含めることはできません。  コンテンツは、地域の法律や文化の規範により、特定の国/地域では機密性の高い、または不快と見なされる場合があります。  さらに、拡張機能と関連するメタデータには、レース、民族、国籍、言語、性別、年齢、障がい、障がい者、性的指向、他のソーシャル グループの資格、またはメンバーシップに基づいて、暴力、暴力、または暴力を支持するコンテンツを含めることはできません。  

### 2.6 ドラッグ、ドラッグ、ドラッグ  

内線に、過剰または無責任な、またはドラッグの過剰または無責任な使用を促進または美化するコンテンツを含めることはできません。  

### 2.7 成人向けコンテンツ  

お客様の内線には、合理的な人物が性的または性的に明示的と見なすコンテンツを含めることはできません。  

### 2.8 禁止コンテンツ、サービス、およびアクティビティ  

拡張機能は、次の条件に従う必要があります。  

*   拡張機能にコンテンツを含めることはできません。また、オンラインでの支援を促進するサービスを提供したりしることはできません。  オンラインのスポーツには、オンラインのスポーツ、スポーツのスポーツ、または現金などの価値を提供するスキルのゲームが含まれますが、これらに限定されるのではありません。  
*   拡張機能は、支払い壁やログイン制限の回避など、Web サイト コンテンツへの不正なアクセスを提供しることはできません。  
*   拡張機能は、著作権で保護されたコンテンツまたはメディアへの不正アクセス、ダウンロード、ストリーミングを提供、奨励、または有効化しることはできません。  
*   拡張機能は、暗号化を利用しることはできません。  
    
### 2.9 不正なアクティビティ  

拡張機能には、現実世界での不正な活動を奨励、助長、または美化するコンテンツや機能を含めることはできません。  

### 2.10 過度な不適切な不適切なコンテンツ  

*   拡張機能に過剰または不当な不適切な言い方を含めることはできません。  
*   お客様の内線には、妥当な人物が難読と見なすコンテンツを含めることはできません。または表示しることはできません。  
    
### 2.11 国/地域固有の要件  

拡張機能の対象となる国/地域で不快なコンテンツは許可されません。  コンテンツは、地域の法律や文化の規範により、特定の国/地域で不快と見なされる場合があります。  特定の国/地域で不快に思えるコンテンツの例を次に示します。  

**中国**  

*   禁止されている性的コンテンツ  
*   係争地域または地域の参照  
*   適用される地域の法律に基い、不正なコンテンツまたはサービスへのアクセスを提供または有効化する  
    
### 2.12 年齢レーティング  

#### 2.12.1 成熟したコンテンツ  

パートナー センターに拡張機能を [提出する][MicrosoftPartnerCenter]場合は、拡張機能に "Mature" とマークする必要があるコンテンツを表示するかどうかを指定する必要があります。  拡張機能のレーティングを決定する際には、ユーザーが生成したコンテンツや広告を含むアプリ内のすべてのコンテンツと、拡張機能がリンクしているコンテンツについて考慮します。  拡張機能に "Mature" コンテンツが含まれていると指定した場合は、このレーティングの正確性を維持する責任があります。  拡張機能に与えられるレーティングに関係なく、Microsoft Edge アドオン開発者ポリシーのすべてのコンテンツ要件に準拠する必要があります。  

#### 2.12.2 評価の変更  

拡張機能が、割り当てられたレーティングよりも年齢レーティングの高い場合に適している可能性があるコンテンツ \(ユーザー生成、販売、その他の Web ベースのコンテンツなど) を提供する場合は、コンテンツ フィルターを使用するか、既存のアカウントでサインインして、そのようなコンテンツの受信をオプトインすることをユーザーに要求する必要があります。  

### 2.13 ビデオ
