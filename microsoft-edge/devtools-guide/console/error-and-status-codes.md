# コンソール エラーと状態コード  

この参照で、DevTools [コンソール](../console.md)からのエラー メッセージとステータス メッセージを解釈できるようになります。  次の表を使用して、コードをプレフィックス \("x" には 0&ndash;9 桁の数字\) で検索します。

| コードのプレフィックス | 分野 | 説明 |  
|:--- | :--- |:--- |  
| [CSP143xx](#csp-codes) | コンテンツ セキュリティ ポリシー | `Content-Security-Policy` HTTP ヘッダーでブロックされたリソース。  |  
| [CSS31xx](#css-codes) | スタイルシート | *Web Open Font Format * \ (WOFF \) に関連するエラー、および*埋め込み OpenType* \ (.EOT \) フォント ソースとホスト サーバーの問題。  |  
| [HTML1112-1300](#html-codes) | HTML | これらの大部分は、 *ドキュメントモード* や *互換表示* など、Microsoft Internet Explorer の概念に固有のレガシ コードです。  |  
| [HTML1400-1532](#html5-parser-warnings) | HTML5 解析 | HTML5 パーサーがスローした検証警告。  |  
| [HTTP](#http-codes) | サーバーのエラーと状態 | HTTP 要求に応答してリモート サーバーが返すコードです。  |  
| [SCRIPT10xx](#javascript-syntax-errors) | JavaScript 構文エラー | JavaScript ステートメントの1つの構造が、1つ又は複数の構文規則に違反している場合に発生します。  |  
| [SCRIPT50xx](#javascript-run-time-errors) | JavaScript ランタイム エラー | システムが実行できないアクションを、スクリプトが実行しようとした場合に発生します。  |  
| [SEC71xx](#security-codes) | セキュリティ | *混在したコンテンツ* や *追跡防止*など、Microsoft Edge が実施するセキュリティ条件を反映します。  |  
| [SVG560x](#svg-codes) | スケーラブル ベクター グラフィックス |  広範なSVG デバッグは、現在サポートされていませんが、一部のコンソール メッセージは、デバッグ用に提供されています。  |  
| WebGL11_xxx | WebGL | WebGL コンテキストからのエラー メッセージ。  [GLSL エラー](https://msdn.microsoft.com/library/dn611835.aspx)も参照してください。  |  
| [XML5xxx](#xml-codes) | XML | XML の解析と検証のエラー。  |  

## CSP コード  

`Content-Security-Policy` HTTP ヘッダーを使用してブロックされているリソースは、DevTools コンソールを通じて報告されます。また、オプションで、サーバーにレポートを返します。

| コード | メッセージ | 説明 | 推奨される修正 |
|:--- |:--- |:--- |:--- |
| CSP14301 | "\ [ポリシーの種類\] の解析に失敗しました。 \<the reason for canceling the operation\>--ポリシーは無視されます"。 | \(例: script-src、 base-uri\) と指定されたセキュリティ ポリシーの種類 は、該当する理由で失敗し、無視されます。  | 特定の種類の必要なリソースがすべて1つのディレクティブにリスト化されていることを確認してください。  たとえば、`script-src https://host1.com; script-src https://host2.com`では、2番目のディレクティブが無視されます。  次の例では、両方のオリジンを有効として正しく指定しています: `script-src https://host1.com https://host2.com`。  |
| CSP14302 | "\[ソースURL \] にあるディレクティブ \[ディレクティブの種類\] の \[ポリシーの種類\] のソースの解析に失敗しました -- ソースは無視されます。"                                                         | ほとんどの CSP ディレクティブには、1つまたは複数のコンテンツ ソース \(コンテンツを読み込める URL を示します\) が必要です。  このエラーは、解析中に失敗したソース URL を含むディレクティブをもつポリシーを指摘しています。  | 特定のディレクティブで定義されているコンテンツソース \(ほとんどの場合はURL\) を確認してください。これは、特定のポリシーの種類で検索できます。  ソース URL を修正または置換するか、ディレクティブを削除します。  <br /> *\*ポリシーには、default-src ポリシーディレクティブを含める必要があります。これは、独自のポリシーを持っていない場合に、他のリソースの種類のフォールバックです。* |
| CSP14303 | "[ポリシーの種類] ポリシーが空です。" | ポリシーの種類 \(例、HTTP ヘッダーのコンテンツ - セキュリティ - ポリシー\) が空です。  | コンテンツ セキュリティ ポリシーを設定するためのクイックリファレンスは、[http://content-security-policy.com](http://content-security-policy.com)にあります。  |

| CSP14305 | "\[ポリシーの種類\] ソースにあるディレクティブ [ディレクティブの種類] のサポートされていないソース \[ソースURL\] は無視されます。" | このディレクティブにリスト化されているソース \(URL、キー ワード、データ) はサポートされておらず、無視されます。  | ソースサイトのアドレスには、オプションの先行ワイルドカード \(アスタリスク文字、`*`\) を含めることができます。  例、http://`*`.foo.com や mail.foo.com:*。   ホストはスペースで区切られています。  ソースはキーワードにすることもできます \(none、self、unsafe-inline、unsafe-eval はすべてサポートされています\) または、データ \(データ: URIs、mediastream: URIs\)。  |
| CSP14306 | " \[ポリシーの種類\] のディレクティブ \[ディレクティブの種類\] に、ソースが指定されていません。--これは 'none' を使用することと同じであり、この種類のすべてのリソースがダウンロードできなくなります。" | ディレクティブしたり、安全なコンテンツにアクセスするためのソースをリスト化しない場合は、ディレクティブに、ディレクティブで指定された種類のリソースがどのようなものでも、ダウンロードできないようにすることになります。  | ソースには、1つまたは複数のインターネットホスト名、 IP アドレス、また、オプションの URL スキームやポート番号を指定できます。  |
| CSP14307 | "ソース [ソース URL] は、"\[ポリシーの種類\] のディレクティブ \[ディレクティブの種類\] に既に提供されています。 " | 重複しているソース \(URL、キー ワード、データ\) は、このディレクティブにリスト化されているため、無視されます。  | 該当する重複するソースを削除します。  |

| CSP14309 | "\[ポリシーの種類\] の \[ディレクティブ名 \] にある不明なディレクティブ -- ディレクティブは無視されます。" | CSP ポリシーに設定されたディレクティブは、不明であり、無視されます。  | サポートされているディレクティブのリストについては、 [http://content-security-policy.com](http://content-security-policy.com/) を参照してください。  |
| CSP14310 | "\[ポリシーの種類\] の [ディレクティブ名] にあるサポートされていないディレクティブ -- ディレクティブは無視されます。" | 解析中、ポリシーの種類 \(例、`Content-Security-policy-Report-Only` ヘッダー フィールド\) に、ディレクティブが見つかりました。これは、サポートされていないため、無視されます。  サポートされているディレクティブについては、 [http://content-security-policy.com](http://content-security-policy.com/) を参照してください。  | サポートされていないディレクティブを削除します。  **注**: \<meta\> 要素または `Content-Security-policy-Report-Only` ヘッダー フィールドでは、一部のディレクティブがサポートされていません。  |
| CSP14311 | "ディレクティブ \[ディレクティブ名\] は、\[ポリシーの種類\] に提供されています -- 重複するディレクティブは無視されます。" | 解析中に重複するディレクティブが見つかりました。2番目のディレクティブとそのソース式は無視されます。  | 重複するスクリプトを削除します。  |

> ユーザーの信頼できるセキュリティ ゾーンにある Web サイトの場合、Microsoft Edge はスタイルシートの MIME の種類をチェックしません。

## HTTP コード  

HTTPエラーコードは、リクエストへの応答としてリモートサーバーから返されます。  おそらく最もよく知られているのは HTTP404 です。これは、サーバーが統一リソース識別子 URIUniform Resource Identifier \(URI\) で指定されたページやドキュメントを見つけられない場合に返されます。

| HTTP401 | 拒否されました | 要求されたリソースには、ユーザー認証が必要です。  |
