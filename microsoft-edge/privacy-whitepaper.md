## 自動  

Microsoft Edge でオートフィルを使うと、パスワード、支払い情報、住所などのフォーム入力データを保存して、生産性を高めることができます。  サイトにアクセスしてフォームの入力を開始すると、Microsoft Edge では、保存したオートフィルデータをフォームに照合するために、フォームの塗りつぶし情報が使用されます。  Microsoft Edge には、同じようなフォームを作成したときに、以前に保存したフォーム入力データが用意されています。  パスワードとクレジットカード情報は、各パスワードとカードに対して明示的に許可した場合にのみ保存されます。  

住所およびその他のフォームのエントリは、既定で保存されます。  住所およびその他のフォームデータの保存およびオートフィルを無効にするには、[ `edge://settings/addresses` **アドレスの保存と入力**] 設定をオフにします。  

Microsoft Edge からパスワードの保存を求めるメッセージが表示されないようにするには、[ `edge://settings/passwords` **パスワードを保存する**] 設定をオンにしてオフにします。  Microsoft Edge が保存されている既存のパスワードにオートフィルしたくない場合、保存したパスワードを削除したい場合は、[ `edge://settings/passwords` **保存済みのパスワード**] に移動して選択します。  オートフィルデータをすべて削除するには、に移動し、 `edge://settings/clearBrowserData` [**フォームデータ**の自動入力] を選択して、目的の時間範囲を選択し、[**今すぐクリア**] を選択します。  

プロファイルの [同期] をオンにしている場合は、同じ資格情報でサインインしているすべてのバージョンの Microsoft Edge で、オートフィルデータが同期されます。  同期をオンにすると、すべてのオートフィルデータが暗号化された Microsoft サーバーに保存されます。  Microsoft サーバーに保存されているオートフィルデータは、同期目的でのみ使用されます。  自動でのデータの同期を無効にするには、お使いのプロファイルの横と [同期をオフにする `edge://settings/profiles/sync` ] ボタンを選択します。 **Turn off sync**  [オートフィルの同期] をオンにしている場合は、Microsoft Edge へのサインイン時に、デバイスからオートフィルデータを削除すると、同じアカウントでサインインしている他のすべてのデバイスからオートフィルデータが削除されます。  

Web ページにアクセスしてフォームを送信すると、Microsoft Edge でフォームに関する情報 (たとえば、ホスト名とオートフィルエントリの種類のハッシュなど) が送信されます (たとえば、ボックス1はメールアドレスを探し、box 2 はパスワードを探しています。  ユーザーが入力した情報やユーザー識別子は、サービスに送信されません。  この情報は、Microsoft Edge がさまざまな web ページでフォームを正しく識別するのに役立ちます。  データは、保存したオートフィルデータをフォームに照合するために使用されます。  

ゲストモードを使用している場合、オートフィルは利用できません。新しいオートフィルエントリは追加されません。  InPrivate ブラウズでは、Microsoft Edge でオートフィルエントリが提供されますが、新しいオートフィルエントリは追加されません。

## コレクション  

Web 上のサイト、テキスト、画像を収集し、Microsoft Edge でコレクションを使ってコンテンツを整理できます。  すべてのコレクションデータは、デバイス上にローカルに保存され、Microsoft Edge プロファイルごとに整理されます。  コレクションに対して同期を有効にしている場合、すべてのメモやコメントを含めて作成されたコレクションは、サインインした Microsoft Edge のすべてのバージョンで利用できます。  

24時間ごとに、特別なエンティティ抽出テンプレートが存在する、サポートされているサイトのリストがダウンロードされます。  テンプレートは各 web サイトに固有のものです。  コレクションに新しい項目を作成すると、Microsoft Edge は、新しいコレクション項目を作成しているサイトが、サポートされているサイトの一覧にあることを確認します。  サイトが一覧に表示されている場合、Microsoft Edge は、特定のサイトのテンプレートのエンティティ抽出サービスを ping します。  サービスへの要求に関連付けられているユーザー識別子はありません。  テンプレートは、収集されるアイテムに関する名前、価格、評価、プライマリ画像、およびその他のデータを識別しようとします。  新しいコレクションアイテムを作成しようとしているサイトが、サポートされているリストサイトにない場合、Microsoft Edge ではテンプレートはダウンロードされません。  テンプレートによって、デバイス上のすべてのコレクション項目をローカルに作成できます。  コレクションアイテムに関するデータは、コレクションを作成するためにサービスに送信されません。  

デバイスに保存されているテンプレートを削除してキャッシュデータをクリアするには、[参照データのクリア] の横にある [クリア `edge://settings/privacy` **するもの**を選択してください] を選び、目的の時間の範囲と種類を選び、[**今すぐクリア**] ボタンを選びます。 **Clear browsing data** **Clear browsing data now**  キャッシュデータを削除するもう1つの方法は、[目的の期間とデータの種類を選択する] に移動し、[ `edge://settings/clearBrowserData` **今すぐクリア**] ボタンを選択します。  

[**コレクションに対する候補を表示**する] 設定をオンにした場合、コレクションのタイトルを使用して、コレクションのタイトルを使って Microsoft Bing 検索を実行すると、関連する Pinterest のトピックページを見つけることができます。  Microsoft Edge では、コレクションに関するデータは Pinterest に送信されません。  候補を削除して、関心のあるトピックページを検索しないようにするには、[ `edge://settings/privacy` **コレクションでの候補を表示する]** 設定をオフにしてオフにします。  

InPrivate ブラウズまたはゲストモードを使用している場合、コレクションは使用できません。  

## がクラッシュする  

クラッシュレポートなどのオプションの診断データが有効になっている場合は、Microsoft Edge がクラッシュしたり、他の信頼性の問題が発生したりしたときに、診断データが収集されます。  診断データは、Microsoft Edge および他の Microsoft 製品およびサービスの信頼性の問題を診断し、解決するために使用されます。

収集された診断データは、Microsoft Edge で信頼性の問題が発生したときにキャプチャされたデバイスとソフトウェアの状態を含むクラッシュダンプの形式になっています。  クラッシュダンプには、信頼性の問題が発生した時点で行われた事象に関する情報が含まれています。  クラッシュの発生時にアクセスした web サイトや CPU 使用率などの情報が診断データに含まれている可能性があります。  クラッシュ診断データはデバイス上にローカルに保存され、クラッシュレポートが有効になっているときに、暗号化されたリンクを使用して Microsoft に送信されます。  各クラッシュダンプには、デバイス固有の識別子、ブラウザーに固有の resettable 識別子、およびその問題を特定するための追加診断データ (URL、CPU 使用率、ネットワーク使用率など) が含まれています。  その他の診断データは、問題が発生しているデバイスの数や重要度など、信頼性の問題を診断するのに役立てるために、クラッシュダンプにアタッチされます。  

クラッシュダンプは、Microsoft に送信され、最長で30日間、セキュリティで保護された Microsoft サーバーに保存されてから、削除されます。  Windows 10 デバイスで診断データの削除を要求するには、 **Start**  >  [**設定**  >  の開始] の**プライバシー**に  >  **関する診断の & フィードバック**] に移動し、[**診断データの削除**] で [**削除**] を選択します。  発生するクラッシュの種類の数など、集計されたクラッシュ情報は、レポートや製品の改善のために保存されます。  

デバイスのファイルシステムからローカルに保存されているクラッシュ診断データをクリアするには、[ `edge://crashes` **すべてクリア**] ボタンに移動します。  

Windows 10 のクラッシュ診断データ収集をオフにするには、[設定の**開始**] の  >  **Settings**  >  **プライバシー**  >  **診断 & フィードバック**に移動します。  他のすべてのプラットフォームの Microsoft Edge のバージョンについては、「 `edge://settings/privacy` **クラッシュレポートを送信して microsoft 製品を改善**する」および「ブラウザーの設定の使い方についてのデータ」を参照してください。  また、[組織で管理][DeployedgeEnterprisePrivacySettings]されているグループポリシーによって、企業の診断データ収集機能が無効になることもあります。

## デジタル著作権管理およびメディアライセンス  

Web サイトがデジタル著作権管理 (DRM \) で保護されたメディアコンテンツを提供している場合、Microsoft Edge はセキュリティで保護された再生パイプラインを使用して、コンテンツが不適切にコピーまたはアクセスされないようにします。  この機能の一部として、Microsoft Edge では、一意の識別子やメディアライセンスなどの DRM 関連データをデバイスに保存し、コンテンツプロバイダーによって指定されたメディアライセンスサーバーに一意の識別子を送信することがあります。  Web サイトを使用すると、Microsoft Edge は DRM 情報を取得し、そのコンテンツを使用するためのアクセス許可があることを確認します。  このデータは、保護されたコンテンツへのアクセスを検証し、シームレスなメディアエクスペリエンスを実現するために役立ちます。  

Microsoft Edge は、HTML5 サイトの暗号化されたメディア拡張 API \ (EME API \) を使用した DRM をサポートしています。  EME API を使うと、web サイトはコンテンツ暗号化解除モジュール \ (CDM \) と呼ばれる DRM プロバイダーと通信できます。  Microsoft による Widevine などのさまざまな DRM システムは、開発者の CDM 実装でサポートされている可能性があります。  コンテンツプロバイダーは、1つまたは複数の潜在的な DRM システムをサポートし、EME API の機能を利用して、特定のクライアントに使用する DRM システムを決定することができます。  EME privacy の詳細については、「[暗号化されたメディア拡張子のプライバシー][W3cEncryptedMediaPrivacy]」を参照してください。  

Microsoft Edge は、Windows 10 でのみ PlayReady DRM をサポートしています。  PlayReady は、4K ビデオや Dolby Atmos audio などのメディアエクスペリエンスを実現するための DRM 実装です。  Microsoft Edge では、Windows プラットフォームメディアファンデーション Api を使って PlayReady をサポートしています。  保護されたコンテンツへのアクセスを検証するために、Microsoft Edge は、一意の識別子 \ (ID) を使用する Windows 10 オペレーティングシステムを利用し、ID を PlayReady サービスと通信します。  デバイスに保存されている PlayReady の EME、CDM、および browser のすべてのデータは、Microsoft Edge に保存され、管理されます。  PlayReady の詳細については、「[簡単なエンドツーエンドシステム][PlayreadyOverviewSimpleEndSystem]」をご覧ください。  

Microsoft Edge では、Google DRM による Widevine がサポートされています。このオプションは既定でオンになっています。  Microsoft Edge は、Google サーバーからの Widevine の更新プログラムを定期的に取得します。  Widevine の使用には、Google へのコミュニケーションが含まれている場合があります。  ユーザーは、Microsoft Edge で widevine DRM フラグを無効にすることによって、Widevine を使用しないようにすることができ `edge://flags/#edge-widevine-drm` ます。  Widevine には、一意のデバイス識別子を作成して Google に送信する機能があります。  Widevine とプライバシーの詳細については、Google のプライバシーポリシーを参照してください。  

Microsoft Edge は、HTML5 の代わりに一部のサイトで使用される、アドビシステムズ社の Flash Access DRM をサポートしています。  サイトから要求された場合は、Adobe Flash を許可する権限を付与する必要があります。  サイトでは、アドビシステムズ社が Flash Access DRM を使用している場合、Microsoft Edge によって固有のデバイス識別子への Adobe アクセスが提供されます。  ローカルに保存された識別子のインスタンスをクリアしてリセットするには、「クリア `edge://settings/privacy` する**データ**を選択する」の「**クリアするもの**を選択する **」の**チェックボックスをオンにし、[**今すぐクリア**] を選択して、保存されている識別子を削除します。  Adobe Flash DRM の使用を停止するには、にアクセス `edge://settings/content/flash` してください。  

オンラインムービーなどの暗号化された HTML5 メディアへのアクセスを要求すると、Microsoft Edge は、メディアの暗号化を解除するためのライセンス要求を作成します。  使用されている CDM によって、要求 ID を含むライセンス要求が作成されます。  要求がライセンスサーバーに送信されます。  ライセンス要求の一部には、個人を特定できる情報が含まれておらず、ライセンス要求はデバイスに保存されていません。  

メディアライセンスを取得すると、ユーザーとサイトに固有のメディア識別子が作成されます。  ID はサイト間で共有されず、サイトごとに異なります。  再生セッションを識別するために使われるセッション ID は、メディア識別子と共に送信され、メディアの暗号化が解除されます。  メディア識別子は、デバイスにローカルに保存され、コンテンツプロバイダーによって保存されている可能性があります。  

DRM およびコンテンツの保護をすべて無効にするには、[ `edge://settings/content/protectedContent` **サイトで保護されたコンテンツの再生を許可する (推奨)** ] および [**保護されたコンテンツの識別子を許可する (コンピューターの再起動が必要な場合があります)** ] 設定をオフにします。  

*   [**保護されたコンテンツを再生できるように**する] 設定をオフにすると、PlayReady や Widevine などの CDM ベースの DRM システムの再生を無効にします。ただし、FLASH Access DRM などの CDM ベース以外のシステムについては再生できません。  Flash サイトの権限を管理するには、にアクセス `edge://settings/content/flash` してください。  設定を無効にすると、メディアの機能が正常に動作しなくなります。  
*   [**保護されたコンテンツの識別子を許可**する] 設定をオフにすると、FLASH Access DRM の識別子は作成されず、Widevine では Google から定期的に更新が取得されません。  設定をオフにすると、一部のサイトで一部のメディア機能が正常に動作しなくなることがあります。  

## トラッキングしない  

Microsoft Edge で [**追跡**しない] を有効にするには、 `edge://settings/privacy` **"追跡**しない" 要求の設定をオンにして有効にします。  **Do Not Track**機能を有効にした場合、Microsoft Edge は、送信した HTTP、HTTPS、および spdy 参照トラフィック要求と共に、ユーザーがトラッカーを使用しないことを要求するために、お客様がアクセスした web サイトへの DNT: 1 HTTP ヘッダーを送信します。  ただし、 **"Not Track" 要求**の設定を有効にしても、web サイトが自分を追跡できないことは保証されません。  一部のサイトでは、以前の閲覧に基づかない広告を表示することで、要求を受け入れることがあります。  Microsoft Edge では、要求が受け入れられるかどうかは制御されません。  Web サイトが追跡されないようにするには、「 `edge://settings/privacy` **バランスをとる**」または「**厳格**にする」に進んで**トラッキング防止**設定を変更します。  

ゲストモードを使用している場合、Microsoft Edge では [**トラッキング拒否**要求] は送信されません。  InPrivate ブラウズを使用している場合、使用しているプロファイルの **"追跡しない" 要求**の設定がオンになっている場合にのみ、Microsoft Edge がトラッキング要求を送信しませ**ん**。  

## ダウンロード  

Microsoft Edge では、安全かつ安全にファイルをダウンロードできます。  デバイス上のファイルをダウンロードする場所を選ぶことができ `edge://settings/downloads` ます。  SmartScreen が有効になっている場合は、ファイル名や URL などのファイルに関する情報が SmartScreen に送信され、ファイルの評価が確認されます。  評価チェックを使用すると、デバイスの障害となっている既知のマルウェアを誤ってダウンロードすることを避けることができます。  で SmartScreen を有効または無効にすることができ `edge://settings/privacy` ます。  SmartScreen の詳細については、「 [smartscreen](#smartscreen) 」セクションを参照してください。  

以前のダウンロードの履歴を表示するには、にアクセス `edge://downloads` してください。  参照データをクリアしてダウンロード履歴を削除するには、にアクセスして `edge://settings/clearBrowserData` ください。  Microsoft Edge からダウンロード履歴を削除しても、デバイスからファイルが削除されることはありません。  ダウンロードしたファイルをデバイスから削除しても、ダウンロード履歴からファイルが削除されることはありません。  InPrivate ブラウズまたはゲストモードを使用している場合、InPrivate またはゲストウィンドウを閉じると、セッションからのダウンロード履歴は消去されますが、ファイルはデバイス上に保存されます。  

## 拡張機能と Microsoft Edge アドオン  

Microsoft Edge に拡張機能をインストールして、ブラウザーに機能を追加することができます。  Microsoft Edge のアドオン web サイトまたは別の拡張ストアから拡張機能をインストールすると、Microsoft は、拡張機能に関する情報を収集し、開発者と Microsoft が拡張機能の使用方法について理解できるようにします。  Microsoft Edge では、拡張のダウンロード回数や、クラッシュデータなどの実行方法に関する情報など、集計データが収集されます。  Microsoft は、拡張の開発者と、集計されたデータを共有します。  ユーザーからのコメントとレビューは、アドオンの web サイトで公開され、開発者とも共有されます。 Microsoft Edge にサインインしている場合は、Microsoft Edge アドオン web サイトからインストールされた拡張機能が、お使いのアカウントに関連付けられ、拡張の推奨事項が提供されます。  データは、拡張機能の普及を理解するために集計で使われます。   

すべてのサインインした同期バージョンの Microsoft Edge 間で、拡張機能の拡張機能と環境設定を同期する場合は、[ `edge://settings/profiles/sync` **同期をオン**にする] を選びます。   

拡張機能のインストールはオプションであり、いつでも拡張機能をアンインストールするには、にアクセスして `edge://extensions` ください。  拡張機能をインストールすると、どのユーザーデータにアクセスする必要があるかが指定されます。 Microsoft Edge は、拡張機能をインストールする前に許可を求めます。  常に、拡張機能が信頼され、安全であることを確認してから、特定の拡張機能について、開発者のプライバシーポリシーを確認してください。  

拡張機能は、Microsoft Edge 更新サービスを使用して更新されます。  Microsoft Edge は、更新プログラムが利用可能かどうかを確認するために、インストールされている拡張機能の一覧を更新サービスに送信します。  Chrome Web ストアから拡張機能をインストールする場合、要求は、拡張機能の更新プログラムを確認するために、通常の間隔で Chrome の Web ストアに送信されます。  Microsoft Edge に関する拡張機能識別子、拡張バージョン、および情報は、更新プログラムの検索要求に含まれています。  Chrome Web ストアへの要求を停止するには、 `edge://extensions` [**その他のソースからの**拡張機能のアンインストール] に移動します。  

Google Chrome などの他のブラウザーから拡張機能をインポートする場合、microsoft edge アドオンの web サイトで拡張機能を使用できる場合は、microsoft edge アドオンの web サイトから拡張機能が自動的にインストールされ、拡張機能が有効になっていた場合は有効になります。  Microsoft Edge のアドオン web サイトから拡張機能を利用できない場合、Microsoft Edge は、その拡張機能を有効にしたり、Chrome Web ストアに接続したりすることなく、Google Chrome から拡張機能をローカルにコピーしてインストールします。  Microsoft Edge では、拡張機能を有効にして、他のストアの内線番号を許可するためのアクセス許可を求められます。  アクセス許可が付与されている場合、Microsoft Edge では、Chrome Web ストアを使用して、他のストアの拡張機能と拡張機能の更新プログラムをインストールできます。  オプションを有効にするには (または \ を無効にするには)、 `edge://extensions` [**他のストアからの拡張機能を許可**する] 設定に移動して有効にします。

[安全な検索を行うと、検索エンジンへのヘッダー要求に安全なキーワードが追加されます。  Bing は安全なキーワードを読み取り、子に返された検索結果をフィルター処理します。  その他の検索エンジンでも、キーワードによってフィルター処理された結果が返される場合があります。  すべての子検索が収集され、家族の開催者がアクティビティレポートまたは[family.microsoft.com][MicrosoftAccountFamilyMain]で表示できるようになります。  データは、お子様の Microsoft アカウントで収集されるため、適切に管理されている可能性があります。

ゲストモードと InPrivate ブラウズは、web のフィルター処理またはアクティビティの報告が有効になっている場合は無効になります。

## フィードバック  

Microsoft Edge チームは常に顧客をリッスンし、フィードバックを指定します。  Microsoft Edge でフィードバックを提供するには、[**設定] を**選択して、[  >  フィードバックの送信 **] をクリック**し  >  **Send feedback**ます。  プログレッシブ Web アプリ (PWAs) の場合は、[**設定**  >  ] などを選択します。**Microsoft にフィードバックを送信**します。  フィードバックの詳細は入力する必要がありますが、その他の情報はすべてオプションです。  Microsoft Edge プロファイルから検出されたメールは、現在のサイトの最新 URL と関連の診断データと共に事前に入力されます。  診断データには、有効にした Microsoft Edge 機能とブラウザーの使用状況に関するデータが含まれる場合があります。  スクリーンショット、デバイスのファイル、およびブラウザーの記録も、必要に応じて含めることができます。  指定したスクリーンショット、ファイル、録音には、個人を特定できる情報が含まれていることがあります。  このデータは、診断と製品の改善のためだけに使用されます。  

ユーザーフィードバックは、HTTPS を使用して Microsoft に安全に送信され、セキュリティで保護された Microsoft サーバーに保存されます。  Microsoft Edge のプライバシー設定で、メールアドレスと、 **microsoft 製品を改善するために、クラッシュレポートとブラウザーの設定の使用方法に関するデータ**が含まれている場合は、お使いのデバイスのブラウザーインストールに固有の識別子が、お客様のフィードバックと関連付けられます。  診断ログ、録音、添付ファイルを含むすべての診断データは、最長30日間保存されます。  省略可能なスクリーンショットを含む残りのフィードバックデータは、最大15か月間保存されます。  フィードバック項目を含むメールを提供した場合は、フィードバックを削除するように[依頼][MicrosoftConcernPrivacy]します。  

## 位置情報  

Microsoft Edge は、[位置情報 API][W3cGeolocationApiMain]をサポートしています。これにより、web サイトは自分のアクセス許可で位置情報にアクセスできます。  Web サイトが自分の位置情報を求める場合があります。たとえば、最も近い喫茶店を見つけようとしたときです。  Microsoft Edge は、web サイトに自分の場所へのアクセスを許可する前に、常に許可を求めます。  アクセス許可を管理するか、サイトにアクセスできないようにサイトをブロックするには、に移動 `edge://settings/content/location` します。  

アドレスバーの右側に、お客様の所在地が共有されているかどうかが示されます。

   Location  
:::image-end:::  

サイトで自分の場所の共有を許可している場合、Microsoft Edge は、IP アドレスや近くの Wi-fi アクセスポイントなどのローカルネットワーク情報を Microsoft の位置情報サービスに送信します。  Microsoft サービスは、情報を使用して位置情報の調整を行います。  位置情報の推定は、現在の場所を共有することに同意したサイトと共有されます。  Microsoft Edge で、要求元のサイトに Windows 10 のより正確な場所を提供するように指定するには、[**設定**プライバシーの場所] に移動し、[  >  **Privacy**  >  **Location** **このデバイス上の場所へのアクセスを許可**する] をオンにして、[**アプリが位置情報にアクセスすることを許可**します] を選びます。  [**このデバイスの位置情報へのアクセスを許可**する] をオフにして、**アプリが位置情報**の設定にアクセスすることを許可した場合、Microsoft Edge は、要求元のサイトに近い場所を提供します。  この情報は、以前に自分の位置情報を表示することを許可している場合にのみ、要求元のサイトで共有されます。  Windows の場所の設定の詳細については、「 [windows 10 の位置情報サービスとプライバシー][MicrosoftSupport4468240]」を参照してください。  

Microsoft Edge の位置情報サービスへの要求を作成するときに、ランダムに生成された新しい ID が使用されます。  Microsoft Edge の位置情報サービスでは、位置情報の座標をいつでも保存することはできません。  

InPrivate ブラウズでは、InPrivate セッションが開始されたプロファイルの [場所のアクセス許可] の設定が使用されます。  ゲストモードでは、サイトを自分の所在地に付与する前に、常に許可を求められます。  

## ブラウザーデータをインポートする  

Microsoft Edge では、ブラウザーを初めて起動するときに、対話型でシームレスなエクスペリエンスを提供します。  エクスペリエンスの一部として、別のブラウザーから Microsoft Edge にブラウザーデータをインポートするオプションがあります。  インポートの際には、インポートしたデータを保持するか、削除して、新しく始めることができます。  データには、お気に入り、閲覧履歴、オートフィルデータ、拡張機能、設定、その他の参照データが含まれます。  旧バージョンの Microsoft Edge の閲覧データは、新しい Microsoft Edge に更新すると、自動的にインポートされます。  確認のため、Microsoft Edge は、Google Chrome、Mozilla Firefox、Internet Explorer などの他のブラウザー、およびインポート元のブラウザーを、OS で定義されている最も使用されているブラウザーに基づいてインポートします。  データのインポートは、デバイス上でローカルに実行され、ローカルに保存され、参照データを同期することに同意しない限り、Microsoft に送信されません。  

:::image type="complex" source="./media/whitepaper-media/migration.png" alt-text="Import" lightbox="./media/whitepaper-media/migration.png":::
   Import  
:::image-end:::  

Microsoft Edge は、Google Chrome などの別のブラウザーから拡張機能をインポートするときに、Microsoft edge アドオンの web サイトで拡張機能を使用できない場合は、ローカルコピーをインポートし、許可を求めます。  一部の拡張機能の権限が変更されている可能性があります。  拡張機能の権限を確認するには、にアクセスし `edge://extensions` ます。  

の任意の時点で、他のブラウザーからデータをインポートすることもでき `edge://settings/importData` ます。  

## インストールと更新  

Windows や macOS などのプラットフォームで Microsoft Edge をダウンロードしてインストールすることができます。  Microsoft Edge は、更新されたバージョンの Microsoft Edge を最新の状態に維持するため、アップデーターサービスを使用します。  

Microsoft Edge をダウンロードしてインストールすると、お使いのデバイスに関する情報 (リリースチャネル、基本的なハードウェア情報、更新識別子、デバイスに固有の識別子など)、およびお使いのブラウザーに固有の resettable 識別子が、インストールプロセス中に Microsoft に送信されます。  デバイスの IP アドレスは、updater サービスに送信されますが、最後の10進数は、プライバシー保護を追加するための scrubbed です。  各閲覧セッションで、更新されたバージョンの Microsoft Edge をインストールするために、ランダムに生成された新しいトークンが作成されます。  トークンは、どの個人情報にも関連付けられておらず、インストールと更新のプロセスにのみ使用され、updater サービスを改善するためにのみ使われます。  

Microsoft Edge は、インストールと更新の進行状況に関する Microsoft Edge アップデーターサービスに対して ping を実行します。  インストールまたは更新に失敗し、クラッシュレポートが有効になっている場合は、ログが作成され、Microsoft に送信されます。  クラッシュレポートを Microsoft に送信する方法の詳細については、「[クラッシュ](#crashes)の問題」を参照してください。  Microsoft edge のダウンロードの成功について詳しく理解するために、microsoft Edge、インストールの成否、およびインストールの取り消しに関する情報を収集します。  

自動更新は、すべての Microsoft Edge ユーザーに対して既定で有効になっています。  すべてのプラットフォームで、Microsoft Edge では、起動時に更新プログラムがあるかどうかがチェックされます。  MacOS デバイスでは、microsoft 製品の更新プログラムも定期的にチェックされます。  組織では、追加のコントロールと構成を利用できます。  追加のコントロールと構成の詳細については、「[更新][DeployedgeUpdatePoliciesUpdate]」を参照してください。

Microsoft Edge では、管理者によって定義された場所からポリシーを使ってサイトの一覧をダウンロードし、IE モードで開く必要があるサイトを決定するファイルをキャッシュします。  Microsoft Edge は、Windows や IE 11 の設定に応じて、ユーザーがどのサイトに移動したか、パフォーマンスデータ、信頼性データ、機能の使用状況データなどの IE モードの使用に関する診断データを収集します。  Windows 10 では、診断データは Windows 診断データの設定に従って収集されます。  Windows 8.1 の場合、web サイトの情報は、ユーザーが IE で [今後フリップ] または [おすすめサイト] 機能をオンにした場合に収集されます。  IE モードは、Microsoft Edge のプライバシー設定では、同じデータ収集の設定に従っていない可能性があります。

## 割り込み広告  

ブラウジングの操作性を向上させるために、Microsoft Edge では、邪魔または誤解される広告を表示するサイトでの読み込みからのアドバタイズをブロックします。  広告ブロックが有効になっている場合、Microsoft Edge は、邪魔または誤認の広告を表示し、デバイスにローカルに保存されている最新のサイトの一覧を定期的に Microsoft サーバーからダウンロードします。  ダウンロード要求にはユーザー識別子は含まれません。  リストにあるサイトにアクセスした場合、Microsoft Edge はそのサイトのすべての広告をブロックし、メッセージが表示され `Ads blocked` ます。  サイトの広告を許可するには、 `edge://settings/content/ads` 設定を取得して変更します。  広告ブロック機能を使用してサイトのリストをダウンロードする以外に、広告ブロック機能は、Microsoft に追加情報を送信したり、web の閲覧中に Microsoft から追加情報を要求したりすることはありません。  

## ジャンプリスト  

Microsoft Edge のジャンプリストを使うと、タスクバーの Microsoft Edge アイコンを右クリックして、最近閉じたサイトを簡単に見つけることができます。  最後の3つの閉じたタブは、各プロファイルのローカルに保存されます。  Windows 10 のジャンプリストからサイトを削除するには、それぞれを右クリックします。  最近閉じたタブをジャンプリストに表示したくない場合は、ブラウザーを閉じるときに、参照データをクリアするか設定を変更して、 `edge://settings/privacy` 閲覧データをクリアすることができます。  InPrivate ウィンドウを使用している場合、Microsoft Edge では、閉じたタブ情報はジャンプリストに追加されません。  ゲストモードを使用している場合、ジャンプリストは使用できません。  参照データのクリアの詳細については、「 [Microsoft Edge でのブラウザーの履歴の表示と削除][MicrosoftSupport10607]」を参照してください。  

## ネットワーク時間  

Microsoft Edge は、Microsoft ネットワークタイムサービスを使用して、タイムサーバーなどの外部ソースから時間を追跡します。  ランダムな間隔で、または Microsoft Edge が期限切れの SSL 証明書を検出した場合、microsoft Edge は、信頼できるソースから時刻を取得するために Microsoft に要求を送信することがあります。  Microsoft Edge がシステムクロックを正しく認識できない場合、要求はより頻繁に発生します。  システムクロック精度が著しくは、ユーザーがオペレーティングシステムの時刻を変更したときに、正しいタイムゾーンと競合している場合に発生します。  Microsoft ネットワークタイムサービスは、協定世界時 (UTC) を取得するために使用されます。  要求には cookie またはユーザー id が含まれず、データは記録されません。

Microsoft Edge では、ユーザー中心の新しいタブページエクスペリエンスを提供します。これには、 [Bing][|::ref2::|Main]を使用した検索ボックス、最も頻繁にアクセスするサイトのクイックリンクタイル、および Microsoft News や Office 365 の関連コンテンツが含まれます。  [カスタマイズ] ボタンを選択して、新しいタブページの外観をカスタマイズします。  新しいタブページの環境設定が各プロファイルに設定され、デバイスにローカルに保存されます。また、環境設定はデバイス間で同期されません。

### Microsoft ニュース  

操作や環境設定に合わせてコンテンツを調整するために、Microsoft Edge の新しいタブページでは、デバイス上でランダムに生成される識別子を持つ cookie を保存します。  Scrubbed バージョンの IP アドレスを使って、コンテンツを一般的な地域に合わせて調整することもできます。  Cookie は、でクリアされるまでデバイス上に保持され `edge://settings/siteData` ます。  

[Microsoft のプライバシーダッシュボード][MicrosoftAccountPrivacy]の[広告設定][MicrosoftAccountPrivacyAdSettings]に移動し、[**ブラウザーで個人用広告を表示**する] 設定をオフにして、広告がパーソナライズされないようにします。  [ユーザー設定 **] ボタン**を選択  >  **Custom**し、[**クイックリンクの表示**] の設定をオフにして、クイックリンクタイルをオフにすることもできます。  Microsoft Edge では、ローカルの閲覧履歴を使用してクイックリンクタイルをカスタマイズし、新しいタイルを削除または作成することができます。  データは、デバイスごとにローカルに保存されます。  

新しいタブページの [検索] ボックスでは、入力した検索クエリに基づいて Bing 検索が実行されます。  検索候補と結果を自動的に提供するために、Microsoft Edge では、入力した文字、検索クエリ、IP アドレス、検索識別子が Bing で共有されます。  検索ボックスは、Microsoft Search からの検索結果を提供するグループポリシーで構成され、ドキュメントやイントラネットコンテンツなどの組織の情報を返します。  統合された検索エクスペリエンスを提供するために、Microsoft Edge は cookie をデバイスにローカルに保存します。  

Microsoft アカウントで Microsoft Edge にサインインしている場合は、 [microsoft のプライバシーダッシュボード][MicrosoftAccountPrivacyAdSettings]から、新しいタブページに関連付けられている閲覧アクティビティを管理することができます。  

Microsoft Edge では、新しいタブページの使用方法に関する診断データが収集されます。たとえば、**クラッシュレポートを送信して Microsoft 製品を改善し、ブラウザー設定の使用方法につい**てのデータをオンにしている場合は、[クイックリンクタイル] をクリックし `edge://settings/privacy` ます。  このブラウザーでは、microsoft News ページを Microsoft に使用した場合の診断データが送信され、ニュースコンテンツとのユーザー操作の理解や Microsoft 製品の改善に役立てることができます。  [新規] タブページで [カスタマイズ] ボタンを選択して、Microsoft ニュースのコンテンツをオフにすることができます。  ニュースデータは、HTTPS を使用して Microsoft に送信され、最大13か月間保存されます。その後、集計されて匿名で作成されます。  

新しいタブページでは、カスタムイメージを背景として設定することもできます。  画像はデバイス上にローカルに保存され、画像を削除するか、または新しい画像をアップロードすることによって削除される可能性があります。  画像に関する情報は Microsoft に送信されません。  

### Office 365  

職場または学校のアカウントを使用して Microsoft Edge にサインインしている場合、組織では、新しいタブページのページコンテンツのオプションとして Office 365 を有効にすることができます。  この機能は、現在商用のお客様のみがご利用になれます。また、 [Microsoft オンラインサービス利用規約 (OST)][MicrosoftLicensingProductProducts]で管理されます。  Office 365 のプライバシーに関する詳細については、「[エンタープライズ向けの Microsoft 365 アプリのプライバシーコントロールの概要][DeployedgePrivacyOverviewControls]」を参照してください。  

InPrivate ブラウズとゲストモードでは、別の新しいタブページのエクスペリエンスを提供します。  

## On startup (起動時)  

Microsoft Edge では、前の閲覧セッションから最後に開いたタブを開くことで、終了した場所を参照することができます。  [続行する] の設定を**オフ**にした場合、 `edge://settings/onStartup` 以前のセッションの情報 (セッション cookie を含む) は、以前のセッションからタブを復元して、自分がアクセスしたサイトにサインインしたままにしておくことができます。  ブラウザーを閉じるたびに [データの閲覧] をオフにすると、指定したデータは削除されますが、次のセッションの場合は URL がその**まま残り**ます。  

また、スタートアップ時に特定のページを開くように Microsoft Edge を設定することもできます。  指定したページは、デバイスにローカルに保存され、プロファイルに固有のものになります。  の [設定の同期] をオンにした場合 `edge://settings/profiles/sync` 、指定したページは、サインインしているすべてのバージョンの Microsoft Edge で同期されます。  

InPrivate モードとゲストモードのタブは、起動時に復元されません。  

## パスワード モニター
Microsoft Edge は、web 上の安全を維持するために取り組んでいます。  個人情報を公開し、セキュリティで保護するために、Microsoft Edge にサインインしている場合、資格情報がサードパーティのデータ漏洩で公開されているかどうかが、パスワードモニターによって通知されます。  パスワードモニターが有効になっている場合、保存されている資格情報は、デバイス上でローカルにハッシュおよび暗号化され、HTTPS 経由で Microsoft サーバーに送信され、既知の違反資格情報の暗号化されたリストと比較されます。  サインインしたアカウント識別子は、ハッシュされて暗号化された資格情報と共に、パスワードモニターサービスに安全に送信されます。  既知のブリーチの資格情報の一覧で資格情報が見つかると、microsoft Edge のバージョンに対して暗号化された返信が送信され、ユーザーの資格情報がハッキングまたは違反の一部として検出されたことを通知します。  チェックが完了した後、Microsoft サーバーにデータが保存されることはありません。   

この機能は、Microsoft Edge にサインインしているユーザーのみが使用できます。  Microsoft Edge では、パスワードモニターを有効にするためのアクセス許可を求められます。  パスワードモニターのオンとオフを切り替えるには、に移動 `edge://passwords` します。

## 支払い  

Microsoft Edge では、ブラウザープロファイルに支払い情報を保存して、閲覧中に必要に応じて情報を使用して支払いフォームに自動的に入力することができるため、生産性を高めることができます。  同様の支払いフォームに遭遇した場合、Microsoft Edge では、保存された情報を使用してフォームに入力することができます。  クレジットカードおよびその他の支払い情報は、明示的なアクセス許可のみで保存されます。  

Microsoft Edge では、支払いオートフィルが有効になっている場合、支払い情報を保存するかどうかを確認するメッセージが表示されます。  情報は、デバイス上でローカルに暗号化されています。  の保存されている支払情報は、削除することができ `edge://settings/payments` ます。  保存された支払い情報を削除すると、情報はオートフィルの候補として表示されなくなります。  の機能を無効にして、支払い情報を保存しないようにすることもでき `edge://settings/payments` ます。  

Microsoft Edge では、以前にオートフィルを使用して保存した支払い情報で購入の支払いを行うことで、PaymentRequest API がサポートされています。  PaymentRequest API を使用すると、販売業者は以下の情報を要求することができます。クレジットカード番号、クレジットカードの有効期限、氏名、請求先住所、メールアドレス、電話番号、および配送先住所。  API は、ユーザーがクレジットカード情報を保存したことを販売業者に通知しますが、お客様が同意するまでは、その商人との情報は共有しません。  の機能をオフにすることができ `edge://settings/privacy` ます。  

Microsoft アカウントに支払い情報を保存したことがある場合は、ブラウザーのオートフィルでも使用できます。  Microsoft アカウントに保存されている支払情報は、デバイス間で同期されます。  以前に Xbox または Microsoft ストアの購入を行ったことがある場合は、Microsoft アカウントに支払い情報が既に保存されている可能性があります。  支払いによるオートフィルでは、Microsoft アカウントのカードがマスクされ、2要素認証後にのみ完全に公開されます。  マスクは、支払い情報を取得するときに追加のセキュリティを提供します。  

## 広告、検索、ニュース、その他の Microsoft サービスをカスタマイズする  

個人設定を許可した場合、Microsoft Edge チームは Microsoft Edge の閲覧履歴を収集して使用し、 [Bing][|::ref3::|Main]、microsoft ニュース、その他の microsoft サービスのエクスペリエンスと広告をカスタマイズします。  [個人用設定] では、より関連性の高い有用な検索結果、広告、ニュースのコンテンツが提供されます。  たとえば、Microsoft Edge チームが、特定のストアでショッピングを希望している参照に基づいて推測された場合、表示される広告は、特定のストアの製品に合わせてカスタマイズされる場合があります。  同様に、旅行用ブログをよく見ていて、旅行の記事を読んでいる場合、ニュースフィードには旅行に関するより関連性の高いニュースコンテンツが含まれていることがあります。  
 
この機能は、子以外の Microsoft アカウントを持つユーザーのみが利用できます。  この機能は、Microsoft Edge に職場または学校のアカウントでサインインしているユーザーには使用できません。  

次の4つの条件がすべて満たされている場合にのみ、閲覧履歴が収集されて、個人用設定に使用されます。  

*   ユーザーは、非子 Microsoft アカウントにサインインしています。 
*   ユーザーは、パーソナル化のためのデータの収集および使用を許可しています。  
*   組織によって管理されているユーザーのグループポリシー (雇用主、学校など) は、個人用設定を許可します。  
*   ユーザーは、ゲストモードまたは InPrivate ブラウズでブラウザーを使用していません。  

閲覧履歴とその他の関連データが HTTPS 経由で転送され、Microsoft アカウントの情報に添付されます。  閲覧履歴は、セキュリティで保護された Microsoft サーバーに保存されます。  [ [Microsoft のプライバシー] ダッシュボード][MicrosoftAccountPrivacy]に移動して、共有されている閲覧履歴を表示したり、削除したりすることができます。  閲覧履歴は、最大45日間、セキュリティで保護された Microsoft サーバーに保存されます。  45日以降、データは削除され、個人設定には使用されません。  

[Microsoft のプライバシーダッシュボード][MicrosoftAccountPrivacy]の[広告設定][MicrosoftAccountPrivacyAdSettings]では、個人向け広告の趣味やオプトアウトを変更することができます。  

> [!NOTE]
> [Microsoft のプライバシーダッシュボード][MicrosoftAccountPrivacy]でパーソナライズされた広告をオフにしても、検索結果とニュースフィードのコンテンツのパーソナライズを行うための閲覧履歴の作成と使用が無効になるわけではありません。  カスタマイズされた検索結果とニュースで Microsoft Edge の閲覧履歴の収集および使用を無効にするには、「 `edge://settings/privacy` **web エクスペリエンスをカスタマイズ**する」で、「**広告、検索、ニュース、その他の microsoft サービス設定をカスタマイズするためにアカウントからの閲覧履歴を使用できるよう**にする」を参照して、Web エクスペリエンスを向上させます。  データの共有を停止すると、Microsoft は閲覧履歴を収集して使用しなくなり、広告、検索結果、ニュースをパーソナライズすることができなくなります。 Microsoft Edge でのパーソナライズの詳細については、「 [Microsoft edge のパーソナライズの広告とエクスペリエンス」を][MicrosoftSupport4532583]参照してください。  

## Print  

Microsoft Edge では、さまざまなデバイスやアプリケーションを使用して web ページ、PDF ファイル、その他のコンテンツを印刷できます。  プリンター、アプリケーション、または PDF に印刷すると、Microsoft Edge は、コマンドとファイル情報をデバイスのオペレーティングシステムに送信します。  この情報は Microsoft に送信されず、デバイスのオペレーティングシステムに送信されたすべてのデータは、印刷が完了または取り消された直後に削除されます。  印刷先を変更するには、に移動 `edge://settings/printing` します。  

Microsoft Print to PDF を使って web ページやファイルを PDF に印刷することもできます。この場合、ファイルに関するデータは Microsoft に送信されません。  PDF ファイルに対して行った注釈は、ファイルにローカルに保存されます。  

## プロファイル  

Microsoft Edge のプロファイルを使用すると、参照データを独立したプロファイルに分けることができます。  1つのプロファイルに関連付けられたデータは、他のプロファイルに関連付けられているデータとは別です。  たとえば、個人用のお気に入りと履歴は、別々のプロファイルで設定した場合、職場のアカウントと同期されません。  

ただし、ユーザーは、パスワードを必要とせずに、Microsoft Edge の既存のプロファイルを簡単に切り替えることができます。  ユーザーが同じデバイスにアクセスできる場合は、現在のプロファイル所有者の許可なしに、同じバージョンの Microsoft Edge で別のプロファイルを作成することができます。  Microsoft Edge の [設定] からプロファイルを削除すると、閲覧履歴、お気に入り、フォームの塗りつぶしデータ、パスワードなど、デバイスに保存されている特定のプロファイルの閲覧データが完全に削除されます。  アカウントに同期されたデータは、依然として Microsoft cloud に保存されている可能性があり、 [microsoft のプライバシーダッシュボード][MicrosoftAccountPrivacy]からクリアされている可能性があります。  

ゲストモードは、新しいプロファイルの一時的なインスタンスです。  サインインしたプロファイルを変更せずに、別のユーザーのデバイスで参照することができます。  お気に入り、閲覧履歴、パスワード、フォームフィルデータなどのゲストモードからのデータの閲覧は、すべてのゲストモードのウィンドウを閉じた後も維持されません。  ダウンロードしたファイルはデバイスに保存されますが、ダウンロードの履歴は削除されます。  ゲストモードでは、他のサイトに自動的にサインインしなくても、web を参照できます。  Microsoft Edge では、ユーザーがゲストモードで閲覧していることを示すために、情報が web サイトに送信されることはありません。  ゲストモードを使用すると、閲覧したブラウザーと web サイトの使用方法に関する診断データを収集する権限が、ゲストモードセッションが開始された Microsoft Edge のプロファイルから取得されます。  すべてのゲストウィンドウが閉じられると、特定のゲストモードセッションの閲覧データはすべて消去されます。  

InPrivate ブラウズは、閲覧履歴、ダウンロード履歴、cookie とサイトデータ、およびフォームのフィルデータが記録されないプライベート参照モードです。  Microsoft Edge では、InPrivate ブラウズで作成されたすべての新しいお気に入りと共に、ダウンロードしたファイルを保存します。  既定では、InPrivate ブラウズの際に、Microsoft は製品の改善のためにアクセスした web サイトに関する情報を収集しません。  学校、職場、またはインターネットサービスプロバイダーが、閲覧アクティビティを見ることができる場合があります。  すべての InPrivate ウィンドウが閉じられると、特定の InPrivate セッションの参照データが消去されます。  Windows 入力方式エディター \ (IME \) キーボードを使用して入力および手書き入力を行うと、データが収集され、言語の認識機能と提案機能が向上することがあります。  InPrivate および通常のブラウズウィンドウで windows IME キーボードを使用しているときに Microsoft によって入力されたデータの入力を停止して、データの入力を停止するには、[設定の**開始**] をクリックして、[  >  個人用**設定**の  >  **Privacy**  >  **& 入力**します。  InPrivate ブラウズの詳細については、「 [Microsoft Edge で Inprivate ブラウズを参照][MicrosoftSupport4533513]する」を参照してください。  
  

## 音声で読み上げる  

Microsoft Edge では、音声読み上げが提供されます。これは、ユーザーに対して web ページのコンテンツを読み取ります。  ページの右クリックメニューから、または [**設定と詳細**] に移動して、音声読み上げを開始します (...)   > **音声**で読み上げることができます。  音声読み上げでは、web ページコンテンツの読み取りに使用できる複数のボイスが提供されます。  Windows 10[にインストール][OfficeSupport4c83a8d8748642f78e462b0fdf753130]されているボイスを windows 10 の設定の [**時刻 & 言語**] セクションの下に使用している場合、以前に使用したボイスはすべてローカルキャッシュに保存され、オフになっている可能性があり `edge://settings/clearBrowserData` ます。  

読み上げを開始すると、Microsoft Edge では[Web SPEECH API][GithubW3cIncubatorCommunityGroupSpeechApi]が使用されます。  選択した音声に応じて、ページのコンテンツは、プラットフォームによって提供されるクライアント側ライブラリ (たとえば、オペレーティングシステムに固有のもの)、または Microsoft の認知サービスを利用するサーバー側ライブラリのいずれかを使用して、テキストから音声に変換されます。  クライアント側のライブラリを使用してコンテンツを音声に変換した場合、Microsoft サーバーに情報は送信されません。  Microsoft の認知サービス (任意の音声名で "Online" と表示される) を使用してコンテンツが音声に変換される場合、テキストはランダムに生成されたトークンと共に Microsoft に送信されます。  変換が完了すると、オーディオファイル内の音声テキストがデバイスに返されます。  すべてのデータは、デバイスから Microsoft およびその逆に転送されている間、暗号化されます。  Microsoft に送信されたテキストと生成された音声ファイルは、変換の発生直後に削除されます。web コンテンツに関するその他のデータは、いつでも保存されることはありません。  

## 新機能のリリース  

Microsoft Edge を改善するために、Microsoft Edge チームは常にユーザーから学習します。  学習の一環として、すべてのユーザーが利用できるようになる前に、一部のユーザーが新しい機能を経験する可能性があります。  ランダムに選択されたユーザーのために新しい機能を有効にするために、Microsoft Edge では、オペレーティングシステム、チャネル、バージョン、国または地域、その他のデバイス構成データについて、必要な情報を定期的に Microsoft Edge 構成サービスに送信します。  データは、ブラウザーに固有の resettable 識別子で送信されます。  データは HTTPS 経由でサービスに転送されます。  このデータは、新しい機能を有効にし、Microsoft Edge を最新の状態にして適切に実行するための更新プログラムの受信、および Microsoft 製品とサービスの向上のために使用されます。  組織では、追加のコントロールと構成を利用できます。  組織向けの追加のコントロールと構成の詳細については、「 [Microsoft Edge の構成と実験][DeployedgeConfigurationExperiments]」を参照してください。  

ユーザーは、組織で管理または構成されているブラウザーの更新を無効にすることはできませんが、製品の利用状況データが Microsoft に送信されるかどうかを制御するには、「 `edge://settings/privacy` **microsoft Edge の品質向上**のための設定の変更」を参照してください。  

新しい機能が Microsoft Edge と Microsoft サービスに与える影響を理解するために、Microsoft Edge では、ブラウザーに固有の resettable 識別子と、有効になっていた新機能を、Microsoft Edge と Microsoft サービスにエンコードする機能タグを送信します。  新しい機能により、すべてのユーザーに最適なエクスペリエンスと最良のブラウザーを構築することができます。  機能タグは、Microsoft Edge のインストールに固有のものではなく、有効になっているのと同じ機能セットを共有する Microsoft Edge のすべてのインストールで共有されます。  Microsoft Edge では、HTTPS の情報が Microsoft サービスに送信されます。  ブラウザーは、InPrivate またはゲストモードで参照すると、情報を送信しません。  データが送信されないようにするには、「 `edge://settings/privacy` **クラッシュレポートを送信して Microsoft 製品を向上させる**」に進み、ブラウザーの設定の使い方についてのデータを有効にします。  ブラウザー固有の識別子をリセットする方法の詳細については、「[ブラウザーセクションの使い方に関する診断データ](#diagnostic-data-about-how-you-use-the-browser)」を参照してください。  

## ナビゲーションエラーを解決する  

Microsoft Edge で、接続のタイムアウト、証明書エラー、または、専用ポータル (ホテルや空港などの Wi-fi ネットワークなど) が原因で発生する可能性があるネットワークの問題が検出された場合、Microsoft Edge は、応答コードへの要求を送信 `http://edge.microsoft.com/captiveportal/generate_204` し、確認します。  要求が別の URL にリダイレクトされた場合、Microsoft Edge では、サインインページであることを前提として、新しいタブに URL が表示されます。  専用ポータルの検出ページへの要求は、ステートレスサービスであり、要求は記録されず、cookie は送信または保存されません。  Windows プラットフォームでは、Microsoft Edge は Windows 専用ポータルサービスを使用します。  それ以外の場合は、Microsoft Edge の専用ポータルサービスが使用されます。  サービスを無効にするには、[ `edge://settings/privacy` **ナビゲーションエラーの解決に役立つ web サービスを使用**する] 設定に移動してオフにします。  

## サインインと Id  

Microsoft Edge にサインインすると、ブラウザーの生産性を高めるための追加機能が提供されます。  シームレスにサインインするには、Microsoft Edge を初めて起動したときに、オペレーティングシステムから id を検出しようとします。  Microsoft Edge でオペレーティングシステムから id が検出された場合、Microsoft Edge にサインインしたままにしたくない場合は、「サインアウト」または「 `edge://settings/profiles` プロフィールの削除」を参照してください。  Microsoft Edge でオペレーティングシステムから id が検出されない場合は、から Microsoft Edge にサインインすることができ `edge://settings/profiles` ます。  

新しい id がオペレーティングシステムに追加されていて、Microsoft edge プロファイルに現在 id がない場合、Microsoft Edge は特定の id をプロファイルに追加します。  Microsoft アカウントまたは職場または学校のアカウントを使用して Microsoft Edge にサインインしていて、Windows 10 プロファイルで id を持っていない場合は、サインイン中に windows 10 に追加しないように指定していない限り、特定のアカウントが Windows 10 プロファイルに追加されます。  

サインインしたプロファイルでは、Microsoft Edge を初めて起動したとき、またはブラウザーにサインインしているときに、明示的なアクセス許可なしでデータの同期が開始されることはありません。  

Microsoft Edge にサインインすると、シングルサインオンが有効になります (Bing などの特定の web サイトに自動的にサインインされます)。また、同期などの識別機能を備えたその他の操作を行うことができます。 Microsoft Edge にのみサインインし、 [Bing][|::ref4::|Main]などの microsoft の他のサイトではサインインしない場合は、特定のサイトからサインアウトすることができます。  Microsoft Edge は、Microsoft Edge が、今後の訪問のために特定のサイトにサインインしないようにするためのサインアウト cookie を作成します。  ユーザー名とパスワードを入力するか、cookie を消去して、特定のサイトにもう一度サインインすることができ `edge://settings/privacy` ます。  参照データのクリアの詳細については、「 [Microsoft Edge でのブラウザーの履歴の表示と削除][MicrosoftSupport10607]」を参照してください。  

Id が Microsoft Edge に関連付けられないようにするには、Microsoft edge プロファイルを削除するか、Microsoft Edge からサインアウトします。  デバイスから Microsoft Edge プロファイルに関連付けられているすべてのデータを削除するには、Microsoft Edge プロファイルを削除する必要があります。  すべてのデータを削除しても、id に関連付けられていた同期済みのデータは削除されません。  

MacOS の Microsoft Edge での id は、Microsoft アプリ間で共有されます。  共有 id を使用すると、デバイスで別の Microsoft アプリにサインインしている場合に、資格情報を個別に入力しなくても、Microsoft アプリにサインインすることができます。  MacOS では、別の Microsoft アプリの認証の状態に基づいて Microsoft Edge に自動的にサインインすることはできません。  Microsoft Edge にサインインしようとすると、デバイス上の別の Microsoft アプリの資格情報を使用して Microsoft Edge にシームレスにサインインすることができます。  同様に、microsoft Edge にサインインしたときに、他の Microsoft アプリにサインインしようとすると、資格情報をもう一度入力しなくても、デバイス上の他の Microsoft アプリにサインインするために Microsoft Edge の資格情報が使用されることがあります。  

ゲストモードまたは InPrivate ブラウズを使用している場合、Microsoft Edge にサインインすることはできません。  

## SmartScreen  

SmartScreen は、web を安全に参照できるように設計されています。  Web サイトにアクセスするか、ファイルをダウンロードすると、SmartScreen は URL またはファイルの評価を確認します。  SmartScreen は、サイトまたはファイルが悪意のあるものであると判断すると、サイトへのアクセスやファイルのダウンロードをブロックします。

:::image-end:::  

Web を参照すると、SmartScreen は、上位のトラフィック、危険、または不明として web サイトやダウンロードを分類します。  主なトラフィックとは、SmartScreen によって信頼されていることがわかっているサイトです。  危険とマークされているサイトに移動しても、SmartScreen でサイトへのアクセスが即座にブロックされます。  不明なサイトに移動すると、SmartScreen は、サイトにアクセスする必要があるかどうかを判断するために評価を確認します。  

SmartScreen では、3種類の評価チェックが使用されます。  まず、SmartScreen はローカルリストに対してアクセスしたサイトの URL をチェックして、サイトが上位トラフィックの一部であるか、既知の危険なサイトであるかを判断します。  トラフィックの上位のサイトにアクセスすると、SmartScreen は SmartScreen サービスに URL を送信しません。  URL が危険なサイトのローカルリストにある場合は、SmartScreen はブロックします。これにより、悪意のある web コンテンツの一部が読み込まれなくなります。  Microsoft Edge は、最新の上位トラフィックと危険なサイトの一覧をデバイスに定期的にダウンロードします。  2番目の URL チェックは、SmartScreen サービスの非同期評価チェックです。  SmartScreen は、上位トラフィックとして分類されていないすべての Url でチェックを実行します。  Microsoft Edge は、サイトに関する URL、関連する情報、デバイスに固有の識別子、およびそのサイトの安全性を判断するための一般的な場所情報を SmartScreen サービスに渡します。  Microsoft Edge によって提供される情報により、サービスは新しい危険なサイトを特定し、最新のセキュリティ上の脅威を常に把握することができます。  URL チェックの結果はデバイスにローカルに保存され、ブラウザーセッションの終了時に自動的にクリアされます。  SmartScreen サービスへのすべての要求は HTTPS 暗号化で作成されます。  

最後に、SmartScreen は、デバイスに害を与えるのを防ぐためにダウンロードされたファイルをチェックします。  ダウンロードが完了すると、SmartScreen は、バイナリファイルの評価チェックを同期的に実行します。  Microsoft Edge は、ファイルハッシュ、ファイル名、ダウンロード URI、デバイスに固有の識別子などのファイルに関する情報を、評価チェックを実行するために SmartScreen に送信します。  すべての SmartScreen 要求は HTTPS 暗号化で作成されます。  SmartScreen サービスはチェックの結果を返します。これにより、ファイルが完全にダウンロードされるかどうかになります。  結果はデバイス上にローカルに保存されます。  

SmartScreen サービスは、評価チェックに関するデータを保存し、既知の悪意のある Url とファイルのデータベースを構築します。  データはセキュリティで保護された Microsoft サーバーに保存され、Microsoft セキュリティサービスにのみ使用されます。  データは、何らかの方法で特定したり、ターゲットを指定したりするために使用されることはありません。  閲覧キャッシュをクリアすると、ローカルに保存されているすべての SmartScreen URL データが消去されます。  ダウンロード履歴を消去すると、ファイルのダウンロードに関するローカルに保存されている SmartScreen データは削除されます。  

Microsoft Edge では、SmartScreen は既定でオンになっています。  SmartScreen を無効にするには、 `edge://settings/privacy` および [**サービス**] の順に移動して、 **Microsoft Defender SmartScreen**の設定を無効にします。  この設定は、デバイス上の Microsoft Edge のインストールに関連付けられているすべてのプロファイルで同じです。  この設定は、デバイス間で同期されません。  この設定は、InPrivate ブラウズとゲスト モードに適用されます。  組織で設定されたグループポリシーでデバイスを管理している場合は、Microsoft Edge に設定が反映されます。  設定を表示するには、に移動 `edge://settings/privacy` します。  SmartScreen の詳細については、「 [smartscreen: FAQ][MicrosoftSupport17443]」を参照してください。  

必要に応じて、SmartScreen は、ダウンロードしたファイルの Url をチェックして、望ましくない可能性のあるアプリとして分類されているかどうかを確認します。  望ましくない可能性のあるアプリをブロックすると、より生産性、パフォーマンス、および楽しいの Windows エクスペリエンスを提供できます。  この設定は既定で無効になっており、Windows 10 デバイスでのみ使用できます。  この機能を有効にするには、[望ましくない可能性のあるアプリのブロック] 設定をオンにして有効にし `edge://settings/privacy` ます。 **Block potentially unwanted apps**  不要なアプリが分類される可能性がある場合の詳細については、「望ましくない可能性のある[アプリケーション (PUA)][WindowsSecurityThreatProtectionIntelligenceCriteriaPotentiallyUnwanted]」を参照してください。  設定の構成方法の詳細については、「望ましくない可能性のある[アプリケーションの検出とブロック][WindowsSecurityThreatProtectionWindowsDefender]」を参照してください。

## 同期  

Microsoft アカウントを使用して Microsoft Edge にサインインすると、microsoft Edge のすべてのサインインバージョンで参照データを同期できます。  お気に入り、設定、フォームの塗りつぶしのデータ (アドレスなど)、パスワード、拡張機能、およびコレクションを同期することができます。  Microsoft Edge で同期を有効にするには、アクセス許可を許可する必要があります。同期された各データ型は、個別にオンまたはオフにすることができます。  [お気に入り] には、以前に以前に設定したすべてのタブが含まれます。以前のバージョンの Microsoft Edge では、他のお気に入りと共に同期します。  Microsoft Edge 同期の1つのサインインバージョンから、または同期がオンになっている他のすべてのバージョンの Microsoft edge に、お気に入りまたは変更されたお気に入りやその他のデータが削除されました。  同期構成は、で管理することができ `edge://settings/profiles/sync` ます。

同期機能には、デバイスの名前、製造元、モデルなど、同期エクスペリエンスを提供するために必要な追加のデバイス接続と構成データが送信されます。  データは、 [Microsoft device dashboard][MicrosoftAccountDevices]から削除されることがあります。  同期されたお気に入りを管理するには、にアクセス `edge://favorites` します。  その他のデータ型を管理するには、にアクセス `edge://settings/profiles` してください。

InPrivate モードとゲストモードの閲覧データは、Microsoft アカウントと同期されません。  ただし、InPrivate セッション中に作成されたお気に入りは、同期が有効になっている、サインインしたバージョンの Microsoft Edge で同期されます。

## 翻訳する

## 製品とサービスの改善のためにアクセスした web サイト  

検索やその他の Microsoft 製品やサービスの向上を選択した場合、microsoft Edge は、Microsoft にアクセスした web サイトに関する情報を送信します。  診断データは、microsoft アカウントで収集または保存されるのではなく、すべてのユーザーを対象とした Microsoft の製品とサービスを強化するために集計で使用されます。  Microsoft Edge でアクセスしたサイトに関する情報を送信すると、サイトの読み込みが速く、検索結果の関連性が高くなる方法を Microsoft が理解するのに役立ちます。

診断データは送信中に暗号化され、最長30日間は、安全な Microsoft サーバーの raw 形式で保存されます。  診断データの重複を確認して集計すると、生の診断データが削除されます。  重複した診断データと集計された診断データは、継続的に Microsoft 製品とサービスの向上のために最大2年間保存されます。  診断データは Microsoft アカウントから収集されていないか、Microsoft アカウントで保存されていないため、診断データは[microsoft のプライバシーダッシュボード][MicrosoftAccountPrivacy]から表示または削除されないことがあります。  デバイスに関連付けられている診断データの削除を要求する Windows 10 デバイスでは、[設定の**開始**] をクリックして  >  **Settings**  >  **Privacy**  >  **フィードバックを &** 、[**診断データの削除**] で [**削除**] を選びます。  他のプラットフォームでは、診断データは削除されるか、30日以内に完全に集計され、匿名で作成されます。

ご使用のバージョンの Microsoft Edge で WIP がオンになっている場合、ブラウザーはイベント ログを収集し、組織に送信します。  WIP が有効になっている場合は、データの収集を無効にすることはできません。  WIP は、2016年8月以降の Windows 10 のリリースでのみ使用できます。  WIP によってキャプチャされたイベントログの詳細については、「 [Windows 情報保護 (WIP) 監査イベントログを収集する方法][WindowsSecurityInformationProtectionCollectAuditEventLogs]」を参照してください。  

## ありがとうございました。  

Microsoft Edge は、 [Chromium][ChromiumMain] open source プロジェクトと、に記載されているその他のオープンソースソフトウェアによって実現され `edge://credits/` ます。  [Google Chrome のプライバシー][GoogleChromePrivacyWhitepaper]に関する白書は、Chromium open source プロジェクトに関する関連情報を収集するためのソースとして使用されていました。  

Microsoft Edge は、お客様のプライバシーを保護して尊重することを目的としています。  Twitter での[@MSEdgeDev][TwitterMsedgedev]について、または **「設定」とその他 (...)**  >  に移動してフィードバックを送信する[**ヘルプとフィードバック**] をクリックして、[質問やコメントで**フィードバックを送信**] を選択します。
